# DX互助会向け — パッケージ参照プロンプト一覧

Cursor に渡す**自然言語の依頼文**のたたき台です。配布用パッケージの「コピペ元」として使う想定です。

## 使い方・注意

- **正本のシェルコマンド**は [運用コマンド一覧.md](運用コマンド一覧.md) です。エージェントやご自身の環境でパスが異なる場合は、そちらを優先してください。
- 215 ワークスペースの **`.cursor/rules/*.mdc`** が入っていることが前提の文面です。ルールをコピーしない環境では、同じ動きにはなりません。
- **2026-04 時点の見直し**を行いました。全体構成の再確認後、文言は差し替えて構いません。

---

## 1. パートナーからの受信をまとめて取り込む

**プロンプト（そのままコピー可）**

```text
パートナーからのメールなどを確認して。
```

**期待する動き（ルール `partner-email-check.mdc` 想定）**

1. `gmail_to_yoritoori.py`
2. `imessage_to_yoritoori.py`
3. `chatwork_to_yoritoori.py`（設定がある場合）
4. `chrline_sync_to_yoritoori.py`（LINE 個人・グループ）
5. `chrline_open_chat_to_md.py`（オープンチャット差分）

**補足依頼の例**

```text
パートナーからのメールを確認して。Gmail は既読も含めて取りこぼし確認したいので --include-read 付きで。
```

---

## 2. LINE オープンチャットだけ同期する

**プロンプト**

```text
LINE のオープンチャットを同期して。
```

別表現（`open-chat-archive.mdc` トリガーと同義）:

```text
オープンチャットを取り込んで。
Squareのやり取りを保存して。
```

**エージェント側の正**: `line_unofficial_poc/chrline_open_chat_to_md.py`（コマンドは運用コマンド一覧 §4）。

---

## 3. 815 神大家オプチャ（有益情報だけ共有）

**プロンプト**

```text
815オプチャの有益情報を、返信提案とは別枠で共有して。
```

**意図**: 815 は返信対象に含めず、ルールで定めたテーマに沿った情報だけ短く知らせる。

---

## 4. パートナーへメール（または iMessage）送信

**プロンプト例**

```text
（相手名）へメールを送って。送信前に Gmail と iMessage のどちらで送るか確認して。
```

**ルール注意**: `partner-email-send.mdc` どおり、**メールと電話の両方**が YAML にある相手では、経路は推測せず確認が必要な場合があります。

---

## 5. 空室対策メール一括（別系統）

業務用の対話フローは Cursor **コマンド**側に手順があります。

- リポジトリ: `.cursor/commands/vacancy-mail-send.md`（存在する場合）
- シェル正本: [運用コマンド一覧.md](運用コマンド一覧.md) §5

パッケージに含める場合は「コマンド名を指定して実行」と一文添えると安全です。

```text
vacancy-mail-send の手順に従って空室対策メールの送信準備をして。
```
