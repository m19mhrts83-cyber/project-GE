# NotebookLM と Cursor でできること一覧（初心者向け）

NotebookLM MCP で Cursor と NotebookLM をつなぐと、**Cursor のチャットから「NotebookLM に登録したノート（ソース）を前提に質問・調査」**できます。  
ここでは、つないだあとに**何ができるか**をカテゴリ別にまとめます。

---

## 全体の流れ（イメージ）

1. **NotebookLM**（notebooklm.google.com）でノートを作り、PDF・Google Docs・Web・YouTube などをソースとして追加する  
2. そのノートの**共有リンク**を Cursor の**ライブラリ**に登録する  
3. Cursor のチャットで「このノートの内容を前提に調べて」「〇〇について教えて」と頼む  
4. AI が NotebookLM に質問し、ノート内の知識をもとに回答を返す  

→ **「自分の集めた資料を、Cursor の会話のなかで参照させる」** ことができます。

---

## 1. 認証・接続まわり

| できること | 説明 | チャットで言う例 |
|------------|------|------------------|
| **初回ログイン** | Google で NotebookLM にログインし、認証情報を保存する | 「NotebookLM にログインして」「setup_auth を実行して」 |
| **接続状態の確認** | 認証が有効か、サーバーが正常か確認する | 「NotebookLM の接続状態を確認して」「get_health で確認して」 |
| **別アカウントに切り替え・再ログイン** | 別の Google アカウントで使いたい、または認証をやり直したいとき | 「NotebookLM を別アカウントで使いたい」「re_auth して」 |

※ 無料アカウントは 1 日 50 クエリまで。上限に達したら `re_auth` で別アカウントに切り替える方法もあります。

---

## 2. ノートのライブラリ管理

NotebookLM のノートは「ライブラリ」として Cursor 側に登録します。登録したノートだけが、チャットから質問の対象にできます。

| できること | 説明 | チャットで言う例 |
|------------|------|------------------|
| **ノートを追加** | 共有リンクを指定して、ライブラリにノートを 1 件追加する | 「このノートをライブラリに追加して：[リンク]」「NotebookLM に [リンク] を追加して」 |
| **ノート一覧を見る** | 登録済みのノートの名前・トピック・用途などを一覧表示する | 「NotebookLM のノート一覧を表示して」「登録済みのノートを教えて」 |
| **ノートを検索** | 名前・説明・トピック・タグでライブラリ内を検索する | 「〇〇についてのノートを検索して」 |
| **ノートの詳細を見る** | 特定のノートの詳細情報（ID・内容・トピックなど）を取得する | 「〇〇というノートの詳細を表示して」 |
| **「いま使うノート」を切り替える** | 複数ノートがあるとき、どれを基準に質問するか選ぶ | 「いまは React のノートで作業して」「〇〇のノートをアクティブにして」 |
| **ノートの情報を更新** | 表示名・説明・トピック・用途・タグなどを更新する | 「〇〇ノートの説明に △△ を追加して」「トピックを更新して」 |
| **ノートをライブラリから削除** | Cursor のライブラリから外す（NotebookLM 上のノートは消えません） | 「〇〇のノートをライブラリから削除して」 |
| **ライブラリの統計** | 登録ノート数や利用状況などの統計を見る | 「NotebookLM のライブラリの統計を教えて」 |

---

## 3. ノートを前提に「質問・調査」する（いちばんの本題）

| できること | 説明 | チャットで言う例 |
|------------|------|------------------|
| **ノートの内容を前提に質問する** | 登録したノート（または指定したノート）のソースを前提に、NotebookLM（Gemini）が回答する | 「このノートの内容で〇〇について教えて」「NotebookLM で △△ を調べて」 |
| **調べてからコードや文章を書く** | 先に NotebookLM で調べさせて、その結果を踏まえてコード・メモ・資料を書かせる | 「この件は NotebookLM で調べてからコードを書いて」「ノートを参照して要約を作成して」 |
| **ライブラリにないノートに一時的に質問する** | ノートの URL を直接指定して、そのノートだけに質問する（登録しなくても 1 回だけ使える） | 「このリンクのノートで〇〇について聞いて：[URL]」 |

※ 質問のたびに「セッション」が作られ、同じセッションなら会話の流れを踏まえたやりとりができます。

---

## 4. セッションの管理

NotebookLM への質問は「セッション」単位で履歴が残ります。

| できること | 説明 | チャットで言う例 |
|------------|------|------------------|
| **セッション一覧を見る** | いまあるセッションと、メッセージ数・最後の活動時刻などを確認する | 「NotebookLM のセッション一覧を表示して」 |
| **セッションの履歴をリセット** | 同じセッションのなかで、会話履歴だけを消してやり直す | 「このセッションの履歴をリセットして」 |
| **セッションを終了する** | 特定のセッションを閉じる | 「〇〇のセッションを閉じて」 |

---

## 5. トラブル時の操作

| できること | 説明 | チャットで言う例 |
|------------|------|------------------|
| **認証を修復する** | ログインが切れた・エラーが出る場合に、再度ログインして認証を直す | 「NotebookLM の認証を修復して」「re_auth して」 |
| **データを掃除してから再ログイン** | 古いブラウザデータやキャッシュを削除し、ライブラリは残したまま認証からやり直す | 「NotebookLM のデータを掃除して（ライブラリは残して）、もう一度ログインして」 |
| **ライブラリを残して完全クリーン** | 認証データ・キャッシュ・ログなどを削除するが、登録ノート一覧（library）は残す | 「cleanup_data でライブラリだけ残して掃除して」 |

※ データ掃除の前に、**Chrome をすべて終了**しておくと安全です。

---

## 6. よく使う言い回し（まとめ）

- **初回だけ**：「NotebookLM にログインして」
- **ノートを登録**：「このノートをライブラリに追加して：[リンク]」
- **何が登録されているか**：「NotebookLM のノート一覧を表示して」
- **調べてから書く**：「この件は NotebookLM で調べてからコードを書いて」「〇〇についてノートを参照して要約して」
- **状態確認**：「NotebookLM の接続状態を確認して」
- **不調のとき**：「NotebookLM の認証を修復して」

---

## 補足

- **NotebookLM 側の共有設定**：ノートの共有を「リンクを知っている人」以上にしておかないと、Cursor から質問できません。
- **無料枠**：NotebookLM 無料で、ノート 100 個・ソース 50 個/ノート・50 クエリ/日 など制限があります。詳細は [NotebookLM](https://notebooklm.google.com) を参照。
- インストールやログインで困ったときは、同じフォルダの **NotebookLM_MCP_インストール手順.md** のトラブルシューティングを参照してください。
