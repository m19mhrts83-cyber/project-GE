# NotebookLM 用マスタースタイル（cute-illustration）

**用途**: このファイルを **NotebookLM のソースとして追加**し、Studio（Nano Banana Pro）で出すスライド・図解の見た目をデッキ全体で揃える。

- **運用の文脈**: [運用まとめ_会話からの記録.md](運用まとめ_会話からの記録.md) の §3.5  
- **スタイル正本（Cursor）**: `~/git-repos/.cursor/commands/cute-illustration.md`  
- **公開 HTML の見た目**: `docs/css/slides-cute.css`（`*_cute.html` デッキ）

---

## NotebookLM での進め方（短いチェックリスト）

1. ノートに **`slides_outline` や話の土台となる MD** を入れる。
2. **このファイル（本 MD）をソースとして追加**する。インデックス完了を待つ。
3. **チャットの最初**で、「登録ソースのマスタースタイルに従うこと」「**Studio で出す各スライド画像も同一スタイル**」と宣言する。
4. **Studio** で **Nano Banana Pro**（UI の表示名は変わる場合あり）を選び、スライド／ビジュアルを生成する。
5. ブレたら **白背景・水彩風ゆるイラスト・丸ゴシック**であることを一言で指摘して再生成する。

## スタイル要約（表）

| 区分 | 指定 |
|------|------|
| 背景 | 白 `#FFFFFF` |
| 線 | 黒の細いペン線、手描き風のやや不均一 |
| 塗り | 水彩風の淡色（クリーム・薄青・薄オレンジ・薄緑）、ムラあり。べた塗りにしない |
| 人物 | 2〜3 頭身、丸顔、点目・線口。服は青・グレー等シンプル |
| オブジェクト | 線画＋淡色、影なし、フラット |
| レイアウト | 1〜4 列グリッド、「イラスト＋短いテキスト」。見出しは黄／オレンジの帯 or マーカー風 |
| 矢印 | 手描き風・曲線 |
| テキスト | 丸ゴシック風・大きめ。強調は黄マーカー風・太字 |
| 強調記号 | 重要は赤（×）、OK は緑（✓） |
| 禁止 | リアル人物、グラデ、ドロップシャドウ、複雑背景、3D |

## マスタースタイル（全文）— チャットに貼る場合もそのまま使える

以下を **そのまま** NotebookLM のチャットや Studio の指示にコピーしてよい。

```
【スライド・図解の統一スタイル（全枚で厳守）】
目的: 日本のビジネス書・解説本の「ゆるいイラスト」で、初心者向けに優しく説明する。

線: 黒の細いペン線。デジタルだが手描き感があり、やや不均一な線。
塗り: 水彩風の淡い色（クリーム、薄い青、薄いオレンジ、薄い緑）。べた塗りではなくムラのある塗り。グラデーションは使わない。
背景: 白 #FFFFFF。複雑な背景パターンは入れない。
人物: 2〜3頭身のデフォルメ。丸い顔。目は点、口は線などシンプルな表情。髪はシンプルな塊。服は青・グレーなど落ち着いた色。ポーズは指さし・腕組み・PC操作など説明的なジェスチャー。
アイコン・物体: シンプルな線画＋淡い色。影なし。フラット。
レイアウト: 1〜4列のグリッド。各セルは「イラスト＋短いキャプション」。見出しは薄いオレンジや黄色の帯、またはマーカー風の背景。
矢印: 手描き風で曲線的。
テキスト: 丸ゴシック風で大きめ。画像内の長文は避け、短い見出しと箇条書き程度。強調は黄色マーカー風のハイライトと太字。
強調記号: 重要・注意は赤（×）、OK・正解は緑（✓）。

禁止: リアルな人物写真風、グラデーション、ドロップシャドウ、複雑な背景、3D表現。

出力は上記に合わせたスライド1枚分（または指定の図）として。モデルは Nano Banana Pro を使う前提。
```

## Studio 用の短い追い文（例）

```
先に合意したマスタースタイル（白背景・水彩風ゆるイラスト・丸ゴシック）を守って。このスライドだけ生成してください：slide_id ○○。画像内テキストは最小限。
```

（`slide_id ○○` を表の `s1` 等に置き換える。）
