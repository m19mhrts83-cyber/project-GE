## 3. スライド制作の流れ（依頼 → 結果 → 自分の作業）

文字（表・文案）を軸にしつつ、**ビジュアルは NotebookLM**、**公開 HTML は Cursor ＋ git**、という分担。

### 3.1 自分が Cursor に出す指示（例）

- `@` で材料を指定（既存 `slides_outline`、旧 HTML、下書き MD など）。
- `dx-slides-from-outline.md` にある **定型ブロック**をそのまま貼ると揃いやすい。内容の要旨は次のとおり。
  - `docs/_drafts/slides_outline_<トピック>.md` に表を作る（列: `slide_id | 種別 | 見出し | 本文 | 図案 | 話す一言`）。
  - 同じ返答に **NotebookLM 用パック**（ソース一覧・スタイル指示・最初のチャット用プロンプト・Studio 用追いプロンプト案）。
- MCP を使う場合は、ノート選択と `ask_question` で文案を磨く、と明示する。

### 3.2 Cursor／エージェント側で得られるもの

- `slides_outline` の実ファイル化（依頼すればリポジトリに保存）。
- NotebookLM にコピペする文章一式、必要なら補助 MD。
- MCP 利用時: 登録済みソース前提の回答（**ソースの Web 投入は別**）。
- PNG が `docs/assets/...` に揃った後: HTML の `img` 差し替え等を依頼可能。
- 本フォルダ `N1_NotebookLM` を編集したタスクでは **rsync ミラー**。

### 3.3 自分が必ず／基本やること（ブラウザ・手元）

1. NotebookLM（Web）でノート作成・**ソース追加**・取り込み待ち。
2. チャットに「最初のプロンプト」を貼る。
3. **Studio** で Nano Banana Pro 等を使いスライド／画像を生成。
4. 画像を `s1.png` … のように `slide_id` に合わせて **`docs/assets/<デッキID>/`** に保存。
5. ブラウザでローカル HTML を確認。
6. **`git add` / `commit` / `push`**（公開）。

**MCP だけでは完結しないもの**: ソースのアップロード、Studio 操作、画像のダウンロード／スクショ、git push。

### 3.4 ビジュアル・画像スタイルの参照（DX互助会スライド HTML 準拠）

NotebookLM Studio や画像生成で**トーンを合わせる**ときの参照先です。本文デッキの見た目の正本は次の HTML（リポジトリ）です。

| 参照 | パス / URL |
|------|------------|
| ローカル（正） | `~/git-repos/docs/DX互助会向け_コミュニケーション連携と一元管理の取り組み紹介.html` |
| GitHub Pages（例） | `https://m19mhrts83-cyber.github.io/project-GE/docs/DX互助会向け_コミュニケーション連携と一元管理の取り組み紹介.html` |

**カラーパレット（画像・スライド生成用）**

| 用途 | 色（HEX） | メモ |
|------|-----------|------|
| ページ背景 | `#151b2b` | ダークネイビー |
| 本文テキスト | `#eef2ff` | オフホワイト |
| 見出し下線・アクセント | `#4bd2b0` | ミント（主アクセント） |
| 小見出し・表ヘッダ文字 | `#8bc3ff` | ライトブルー |
| リンク | `#7fc9ff` | |
| 注釈ブロック背景 | `#0f2e4f` | 左縦線も `#4bd2b0` |
| 図表・コードの台紙 | `#10182a` | Mermaid / コード塊 |
| 枠線（表・枠） | `#2a3550` | |

**タイポ・レイアウト**

- フォント系: 日本語は **ヒラギノ系サンセリフ**（丸ゴシック寄りの太め見出しでも可だが、HTML 正本はゴシック体）。
- スライド: **余白多め**、1画面の情報は詰めすぎない。見出しは下に **2px 程度のミントのアンダーライン**イメージ。
- 図解: 背景は上記ダーク系、矢印・枠は `#4bd2b0` / `#8bc3ff` の二色使いで統一。

**NotebookLM（Studio / 画像生成）に貼るスタイル指示・たたき台**

```text
スライド用のビジュアルを生成してください。スタイルは次に厳密に合わせてください。
- 全体: ダークモード。背景は深いネイビー #151b2b。本文は明るい灰白 #eef2ff。
- アクセントカラー: ミントグリーン #4bd2b0（見出しの下線、重要な枠、フローチャートの主線）。
- サブアクセント: ライトブルー #8bc3ff（副見出し、図の補助ラベル）。
- 図表のパネル背景: #10182a。区切り線・表の枠: #2a3550。
- 注意書きボックス: 背景 #0f2e4f、左端に太めの #4bd2b0 の縦ライン。
- フォント: 日本語はクリーンなゴシック体。装飾過多にしない。
- トーン: 勉強会資料向けの落ち着いたプロ向け。イラストは必要最小限。
```

（トピックに応じて「可愛い系に寄せる」等は `slides_outline` の図案列と両立させ、上記パレットは維持する。）

### 3.5 ゆるイラスト（cute-illustration）— NotebookLM / Nano Banana Pro でデッキ全体に適用

**目的**: 初心者向けの説明資料。ダーク系（§3.4）とは別系統。**スライドの見た目の正**は Cursor コマンド [`.cursor/commands/cute-illustration.md`](../../.cursor/commands/cute-illustration.md) と、HTML 側の `docs/css/slides-cute.css`（`*_cute.html` デッキ）。

**NotebookLM 用の単体ファイル（ソース追加用）**: **[NotebookLM_マスタースタイル_cute-illustration.md](NotebookLM_マスタースタイル_cute-illustration.md)**  
→ チェックリスト・スタイル表・**マスタースタイル全文**・Studio 追い文の例を **1 ファイルに収録**。重複を避けるため、§3.5 の詳細はそちらを正とする。

**NotebookLM での進め方（要点）**

1. ノートに `slides_outline`・補助 MD を入れる。
2. 上記 **NotebookLM_マスタースタイル_cute-illustration.md** を **ソースとして追加**し、インデックス完了を待つ。
3. チャットの最初で「Studio で出す各スライドも同一スタイル」と宣言。
4. **Studio** で **Nano Banana Pro** を選んで生成。ブレたら **白背景・水彩・丸ゴシック**を再指定。

## 4. 認識しやすいポイント（会話での整理）

- **表ができただけでは公開は未完**（PNG と HTML の `src` の対応が必要な場合がある）。
- **NB を使わない**運用も可能（その場合は `dx-slides-from-outline.md` のブロック B を省略し、SVG 等で回す）。
- **ルールファイル**は `.gitignore` の例外により `n1-notebooklm-mirror.mdc` だけリポジトリに含めている（他の `.cursor/rules` は無視のまま）。
