export GMAIL_TOKEN_PATHS="$HOME/git-repos/215_kamiooya/C1_cursor/1b_Cursorマニュアル/token_estate.json,$HOME/git-repos/215_kamiooya/C1_cursor/1b_Cursorマニュアル/token_m19m.json"

**LINE（CHRLINE sync）**:

```bash
cd ~/git-repos/line_unofficial_poc && .venv/bin/python chrline_sync_to_yoritoori.py --allow-qr-login
```

- プリセット: 既定 `line-default`（**Tcell はグループ「キャラメル管理G」のみ**＋ LEAF グループを同一実行で処理。業務連絡はグループに集約する想定）。Tcell グループのみ `--preset tcell-both`、過去の 1:1 取り込みだけ `--preset tcell-yuki`、LEAF のみ `--preset leaf-grandole`。
- 前提: `LINE_UNOFFICIAL_AUTH_DIR`（**未設定なら** `line_unofficial_poc/.line_auth_local` に自動。クラウド同期パスは非推奨）、必要に応じて `LINE_TCELL_YORITOORI_MD` / `LINE_LEAF_YORITOORI_MD` 等。CHRLINE 再利用時は `savePath/.images/qr_*.png` を実行ルーチンで削除済み。
- セッション失効時は QR 再認証が必要（非公式 API の制約で省略不可）。`line_unofficial_poc/.env` で **`LINE_CHRLINE_AUTO_OPEN_QR=1`** を付けたときだけ QR 画像を自動表示（未設定なら表示せず、ターミナルの `IMG:` パスを手動で開く）。古い保存トークンが残っていれば別ファイルを自動試行して QR 回数を減らす。
- `chrline_sync_to_yoritoori.py` は **既定で QR 再認証を起動しない**（保存トークン無効時は終了）。「やり取り確認」を明示実行するときだけ `--allow-qr-login` を付ける。

**本文取得率の比較検証（defer ON/OFF）**:

```bash
cd ~/git-repos/line_unofficial_poc

# A) 既定（defer ON）
.venv/bin/python chrline_sync_to_yoritoori.py --preset line-default --allow-qr-login --verbose

# B) 比較用（defer OFF）
.venv/bin/python chrline_sync_to_yoritoori.py --preset line-default --allow-qr-login --no-defer-sync-placeholders --verbose

# C) 直近14日で本文取得率を可視化
.venv/bin/python chrline_decode_stats_report.py --since-days 14
```

- `chrline_sync_to_yoritoori.py` は毎回 `LINE_UNOFFICIAL_AUTH_DIR/.chrline_sync_decode_stats.jsonl` に本文取得率を追記（`--decode-stats-file` で変更可）。
- `chrline_decode_stats_report.py` は `defer_sync_placeholders` の ON/OFF 比較と、ターゲット（1:1 / G）別の取得率を表で表示する。

## 4. LINE オープンチャット差分の蓄積

```bash
cd ~/git-repos/line_unofficial_poc && .venv/bin/python chrline_open_chat_to_md.py
```

- ルート YAML: 環境変数 `LINE_OPEN_CHAT_ROUTES_YAML`（未指定時 `open_chat_routes.yaml`）
- **認証はトーク同期と共有**: `chrline_open_chat_to_md.py` も `chrline_sync_to_yoritoori.py` も同じ `LINE_UNOFFICIAL_AUTH_DIR` と `chrline_client_utils.build_logged_in_client` を使う。**別ディレクトリに分けると QR ログインが冗長に増えやすい**。
- **`.tokens/`**: 複数ファイルを更新日時の新しい順に試行。再認証後は `persist_auth_token` で保存され、次回以降 QR なしで起動しやすい。
- **クラウド同期パス**: `LINE_UNOFFICIAL_AUTH_DIR` が OneDrive 等を含むと警告（セッション競合で QR が増えやすい）。未設定なら `line_unofficial_poc/.line_auth_local` 既定を推奨。
- QR・`LINE_CHRLINE_AUTO_OPEN_QR`: 上記「LINE（CHRLINE sync）」と同じ。セッション失効時の QR 自体は非公式 API の制約で省略不可。
- （任意）疎通監視: `line_unofficial_poc/launchd/open_chat_healthcheck_runner.sh` … 失敗時に通知し再ログインを促す。
