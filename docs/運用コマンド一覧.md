### 日常差分（メイン / スレッド専用の分離・推奨）

**方針**: 初回は可能な限りバックフィル（`chrline_thread_bootstrap.py --full` 済み）。日常は **開いていて取得可能なスレッドだけ** 差分を取る。閉鎖済みは state で `closed` と記録し **再参照しない**（YAML の `thread_mids` と MD 上の履歴は残す）。

```bash
cd ~/git-repos/line_unofficial_poc

# メインのみ（6 ルート）— 新規スレッド MID の discover もここ
.venv/bin/python chrline_open_chat_to_md.py --allow-qr-login --no-threads

# スレッド差分（取得可能なものだけ。現状おおよそ 37 件 / YAML 503 件）
.venv/bin/python chrline_open_chat_to_md.py --allow-qr-login --threads-only
```

| state `health.status` | 意味 | 日常差分 |
|---|---|---|
| `ok` | 取得可能・差分監視中 | 対象 |
| `closed` | 閉鎖・参加不可（401 等） | **スキップ**（MD の過去分は保持） |
| `deleted` | 削除済み（404） | スキップ |

見出し形式（新規追記分）: `### 日付｜【メイン】｜グループ名｜受信｜要約` / `### 日付｜【スレッド】｜グループ名｜「スレッドタイトル」｜受信｜要約`  
スレッドタイトルは API から取得し `open_chat_routes.yaml` の `thread_titles` に蓄積。未取得時は MID 先頭8文字。

**既存見出しの一括変換**（旧 `LINEオープンチャット・メイン` 形式 → `【メイン】` 形式）:

```bash
cd ~/git-repos/line_unofficial_poc
.venv/bin/python chrline_migrate_open_chat_headings.py
# 確認のみ: --dry-run
```

**スレッドタイトル一括取得**（取得可能なスレッドのみ・YAML 更新）:

```bash
cd ~/git-repos/line_unofficial_poc
.venv/bin/python chrline_open_chat_to_md.py --allow-qr-login --resolve-thread-titles
# 閉鎖済みも試す: --include-closed-threads（多くは 401/404 で取得不可のことがある）
```

API でタイトルが取れないときは、既存 MD から最古メッセージ要約で推定して見出しを差し替え:

```bash
.venv/bin/python chrline_migrate_open_chat_headings.py --infer-thread-titles --write-yaml-titles --refresh-titles
```


- **`chrline_yoritoori_inbox_fetch.py`** はメイン → スレッドの 2 段を **1 回の QR** で実行。
- 新規スレッドはメイン差分の `--discover-thread-mids-dry-run` で候補検出 → ユーザー確認後 YAML 追記 → 初回のみ `--init --no-main` でバックフィル。
- 閉鎖済みを再スキャンしたいときだけ `--threads-only --include-closed-threads`（通常は不要）。
- 終了時: `# thread diff: YAML登録 503 件のうち取得可能 37 件を対象（閉鎖済みスキップ 466 件）` のような行が stderr に出る。

- **再ログイン**: オープンチャット同期でも `--allow-qr-login` を付けたときだけトークン失効時に QR を開始する。付けずに実行した場合は終了コード 2 で終了するので、意図しない時間帯に QR が出ない。

- ルート YAML: 環境変数 `LINE_OPEN_CHAT_ROUTES_YAML`（未指定時 `open_chat_routes.yaml`）
- **認証はトーク同期と共有**: `chrline_open_chat_to_md.py` も `chrline_sync_to_yoritoori.py` も同じ `LINE_UNOFFICIAL_AUTH_DIR` と `chrline_client_utils.build_logged_in_client` を使う。**別ディレクトリに分けると QR ログインが冗長に増えやすい**。
- **`.tokens/`**: 有効トークンは **1 件のみ保持**（失効分は試行時に自動削除）。再認証後は `persist_auth_token` で保存。
- **クラウド同期パス**: `LINE_UNOFFICIAL_AUTH_DIR` が OneDrive 等を含むと警告（セッション競合で QR が増えやすい）。未設定なら `line_unofficial_poc/.line_auth_local` 既定を推奨。
- QR・`LINE_CHRLINE_AUTO_OPEN_QR`: 上記「LINE（CHRLINE sync）」と同じ。セッション失効時の QR 自体は非公式 API の制約で省略不可。
- （任意）疎通監視: `line_unofficial_poc/launchd/open_chat_healthcheck_runner.sh` … 失敗時に通知し再ログインを促す。

### スレッド MID 補助（メインタイムラインから候補抽出・`open_chat_routes.yaml` 追記）

- 参加していないスレッドは `getJoinedSquareChatThreads` に載らないが、**メインタイムラインの `[relatedMessageId]`** から `getSquareThreadMid` で解決できる（`--discover-from-yoritoori`、bootstrap の discover で既定 ON）。`thread_mids` を YAML に登録するとスレッド専用タイムライン（見出し `【スレッド】`）として取り込める。
- **CHRLINE アプリ版**: Square スレッド API には **9.0.0.3360** 以上が必要（`chrline_client_utils` 既定。上書きは `LINE_CHRLINE_APP_VERSION`）。
- **日常（差分のみ）**: パートナー取り込み統合スクリプトの `--discover-thread-mids-dry-run`（同期後・差分イベントから新規 MID 検出）。
- **初回 bootstrap（履歴スキャン）**: 下記 `chrline_thread_bootstrap.py` または `--discover-only`（yoritoori 由来 discover 含む）。
