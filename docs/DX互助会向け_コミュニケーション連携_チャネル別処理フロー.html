5チャネル別 · 処理フロー概要（1ページ）

各スクリプトの通常の成功パスだけを短いフローチャートにしたものです。分岐・エラー・再認証などは省略しています。実装の正は各 .py の docstring とソースを参照してください。

1. Gmail API

主な実装: 215_kamiooya/C1_cursor/1b_Cursorマニュアル/gmail_to_yoritoori.py

flowchart TB gm1["credentials.json / token（OAuth）"] --> gm2["Gmail API でメール検索・本文取得"] gm2 --> gm3["連絡先一覧.yaml と送信元アドレス照合"] gm3 --> gm4["日時・件名・本文を整形"] gm4 --> gm5["該当フォルダの 5.やり取り.md に追記"] gm4 --> gm6["添付はフォルダ内 添付/ へ保存"]

2. ChatWork API

主な実装: 1b_Cursorマニュアル/chatwork_to_yoritoori.py（リポジトリ外のみの場合あり）

flowchart TB cw1[".env の CHATWORK_API_TOKEN"] --> cw2["ChatWork API でルーム・メッセージ取得"] cw2 --> cw3["連絡先一覧の chatwork_room_id と照合"] cw3 --> cw4["整形（日時・投稿者・本文）"] cw4 --> cw5["該当フォルダの 5.やり取り.md に追記"]

3. iMessage（macOS）

主な実装: 215_kamiooya/C1_cursor/1b_Cursorマニュアル/imessage_to_yoritoori.py

flowchart TB im1["~/Library/Messages/chat.db"] --> im2["SQLite で新着メッセージ取得"] im2 --> im3["連絡先一覧 phones と電話番号照合"] im3 --> im4["日時・本文を整形"] im4 --> im5["該当フォルダの 5.やり取り.md に追記"] im5 --> im6["処理済み ROWID を JSON に保存"]

4. LINE（個人・グループ）

主な実装: line_unofficial_poc/chrline_sync_to_yoritoori.py（差分同期は chrline_sync_delta_poc 系）

flowchart TB ln1["LINE_UNOFFICIAL_AUTH_DIR / .tokens"] --> ln2["build_logged_in_client で CHRLINE 起動"] ln2 --> ln3["同期オペレーション（メッセージ系）を取得"] ln3 --> ln4["プリセット・chatMid で対象トークに絞込"] ln4 --> ln5["dedup（既追記 ID はスキップ）"] ln5 --> ln6["103_Tcell / 104_LEAF 等の 5.やり取り.md に追記"]

5. LINE オープンチャット（Square）

主な実装: line_unofficial_poc/chrline_open_chat_to_md.py

flowchart TB oc1["open_chat_routes.yaml（ルート定義）"] --> oc2["build_logged_in_client（トーク同期と共有可）"] oc2 --> oc3["Square でメイン・スレッドを取得"] oc3 --> oc4["state.json で続きから / dedup で重複除外"] oc4 --> oc5["各 output_md（例: 815 の 5.やり取り.md）へ追記"]

オープンチャット・CHRLINE は非公式 API のため、取得できない本文や再認証（QR）が発生し得ます。詳細は取り組み紹介スライドの「認証まわり」とハンドブックを参照してください。
