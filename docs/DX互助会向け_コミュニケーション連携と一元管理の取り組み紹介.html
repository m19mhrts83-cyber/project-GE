DX互助会向け コミュニケーション連携と一元管理

今日の約束: パートナーとのやり取りを「どこにどう残し」、Cursor で「どう再現するか」— 実例と、持ち帰れるパッケージまでを一本で見る。

1. 概要 — 何をしているか

Gmail に限らず、複数チャネルから届くパートナーとのやり取りを、主としてパートナー別の 5.やり取り.md に時系列で蓄積する運用です。チャネルが増えても、読む場所と取り込みの型を揃えます。

  • Gmail API / ChatWork API / iMessage / LINE（個人・グループ）/ LINE オープンチャット
  • オープンチャットの一部は別 MD（例: 情報収集枠）に分ける場合あり

2. 構築の苦労と工夫（例: LINE 周り）

改善してきたこと（ねらい）

  • 認証の一本化 … LINE_UNOFFICIAL_AUTH_DIR を、トーク同期とオープンチャット取得の両方で共有。同じ chrline_client_utils.build_logged_in_client 経由にし、「スクリプトごとに別ログイン → QR が連発」になりにくくした。
  • ローカル既定の保存先 … 未設定時は line_unofficial_poc/.line_auth_local。クラウド同期直下は競合しやすく QR が増えやすいため、警告する。
  • 保存トークンの順試行 … .tokens/ 内を更新日時の新しい順に試し、有効なら QR なしで起動。
  • 再認証後の永続化 … QR で得たトークンを保存し、次回の手間を減らす。
  • 失効の気づき … 例: launchd/open_chat_healthcheck_runner.sh で疎通チェック＋通知。

苦労した点（割り切り）

  • CHRLINE は非公式 … LINE 側の変更でセッションが切れる。復旧時の QR は「回数を減らす」が現実的な上限。
  • Square（オープンチャット）は取得範囲に依存し、スレッド本文が欠けることもある（付録フローと Beta 運用をセット）。

任意: LINE_CHRLINE_AUTO_OPEN_QR=1 で QR 画像を自動表示（認証回数は減らないが操作負荷は下げられる）。

3. メリット — なぜこの形にするか

  • 記録の一元化 … チャネルが分かれていても、後から読む場所がパートナー単位で固定される。
  • 送信導線の統一 … パートナー宛は yoritoori_send.py に寄せ、下書き（4.送信下書き.txt）と履歴追記の型を揃える。
  • ルールで手順を固定 … 「取り込みはこの順」「送信はこのスクリプト」と Cursor 側に書いておき、依頼のぶれを減らす。
  • Cursor と相性がよい … 蓄積したテキストをコンテキストに載せやすく、次のスライドのような実運用での効きにつながる。

技術的な制約（非公式 API 等）はスライド 2 のとおり。それでも「探す・迷う時間」を減らす設計に価値がある、という整理。

4. 運用で良かった点（実感）

一元化してから、次のような日々の効きを感じています（発表者の感想として）。

  • チャット一言で状況が掴める … 複数チャネルをまたいでも、まとまったログがあるので、Cursor に依頼したときに全体の様子を短い会話で把握しやすくなった。
  • 返信メールの作成がしやすい … 蓄積したやり取りを読ませて、文面のたたき台を一緒に考えてもらう動きがやりやすくなった。
  • 返信が必要なものの整理 … 「確認して」「優先度の高いものを教えて」と頼むと、先に手をつけるべき相手・件が立ち上がりやすい。
  • クイックに着手できる … 重要なメール・やり取りに対して、あちこち探してから始める量が減り、会話の中で「今やること」が明確になる。
  • 探す・迷う時間が減った … 記録先と型が揃っていることで、検索や記憶頼みの負担がだいぶ少なくなった、という実感。

5. ローカル適用パッケージの紹介

同じ「型」を、各自の PC のワークスペースに載せて試せるよう、手順・プロンプト例・ルール案をまとめたパッケージ（MD・スクリプト・.mdc）を用意しています。フォルダ構成は人ごとに違ってよい前提です（ハンドブック §0-1）。

誘い: このパッケージをそのまま真似（パクる）して、自分の環境で動かしてみませんか。完コピでなくても、導線とルールの型だけ持ち帰ってもらえれば十分です。

主な同梱・参照: ローカル適用ハンドブック · パッケージ参照プロンプト一覧 · 運用コマンド一覧

6. ルール（.mdc）— 再現性の中核

Cursor のプロジェクトルールに置くと、依頼文と実際のスクリプト実行がつながりやすくなります。

  • partner-email-check.mdc … Gmail / iMessage / ChatWork / LINE / オプチャ取り込みの導線
  • partner-email-send.mdc … 送信を yoritoori_send.py に統一
  • open-chat-archive.mdc … オープンチャット取り込み導線
  • 815-openchat-no-reply-proposal.mdc … 返信提案と情報共有の分離（815 枠）

流れ（3 ステップ）

  1. ローカルに置く … Cursor で開くフォルダをワークスペースのルートに。MD は docs/ など分かりやすい場所へ。
  2. 参照する … チャットで @ + ハンドブック（ファイル名）を指定し、「この手順に沿ってセットアップして」と依頼。
  3. 環境に合わせる … 運用コマンド一覧のパスを、自分のルート前提で書き換え（または Cursor に書き換え依頼）。

ルール .mdc が無い環境では、プロンプト一覧に書いた「期待する動き」とは一致しません。
