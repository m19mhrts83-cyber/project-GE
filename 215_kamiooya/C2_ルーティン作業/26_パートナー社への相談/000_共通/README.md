# 000_共通フォルダ

パートナー社共通で使うルール・メモを置くフォルダです。

| ファイル | 内容 |
|----------|------|
| Notionタスク_プロジェクト名ルール.md | タスク登録時のプロジェクト名の付け方 |
| Notionタスク_担当設定メモ.md | 担当者（Notionリレーション）の設定メモ |
| 管理会社メール履歴_運用の選び方.md | メール履歴の運用方針 |
| 連絡先一覧.md | パートナー社のメール・電話・LINE（人が見る用） |
| 連絡先一覧.yaml | 同上（Gmail 自動連携スクリプト用） |

パートナー社ごとのやり取り・送信下書きは、**各社フォルダ**に入っています。

---

## Gmail → やり取りの自動連携

パートナー社からの返信メールを、該当フォルダの `やり取り.md` に自動追記する仕組みがあります。

1. **連絡先一覧.yaml** に各社のメールアドレスを登録する
2. **Gmail API** を設定（`C1_cursor/1b_Cursorマニュアル/Gmailやり取り連携_セットアップ手順.md` を順番に実施）
3. `python gmail_to_yoritoori.py` を実行すると、未読メールの送信元を照合して該当フォルダに追記
4. **添付ファイル**は各フォルダ内の「添付」サブフォルダに保存され、やり取り.md に一覧が追記される

## LINE → やり取りの半自動追記（個人LINE・グループ）

個人LINEに公式の送受信 API はないため、**まず公式のトーク履歴エクスポート .txt（推奨）**、短い抜粋は **コピペ**、の二段構えで `5.やり取り.md` に追記します。

### おすすめ: 公式エクスポート .txt → `line_export_to_yoritoori.py`

1. LINE アプリで **トーク履歴の送信／エクスポート**（機種でメニュー名が異なる）し `.txt` を保存。
2. `pip install -r requirements_gmail.txt`（**linelog2py** 含む）済みの `./.venv_gmail/bin/python` で:  
   `line_export_to_yoritoori.py --export-file （パス）.txt --partner Tcell --group --group-label "キャラメル管理G"`  
   形式が合わないときは自動で全文プレーン。強制プレーンは `--plain`。

### 補助: コピペ → `line_to_yoritoori_clip.py`

1. 連絡先を YAML で確認。2. LINE でコピー。3. `line_to_yoritoori_clip.py --partner …`（`--group` 等は同上）。

メール文面は **`5.やり取り.md` を土台に Cursor／`4.送信下書き.txt` で作成し、完成文をコピー**する運用で十分実用しやすいです。

**パートナー別にまとめる運用**: やり取りは **パートナーごとのフォルダ**（例: `103_Tcell`）の **`5.やり取り.md`** に時系列で載せる想定です。  
**LINE のトーク名**は **`連絡先一覧.yaml` の `line:`** に一覧表示どおり書く（`line_mac_open_chat_poc.py` の `--title` 兼用）。複数トークは `line:` に並記するか、専用フォルダを切る。

**任意（実験用）**: `line_mac_open_chat_poc.py` の `--title` には **`line:` の文字列**（または一部）を使うとよいです。バージョンで動かないことがあります。**アクセシビリティ・自動化の許可**が必要。**送信は行いません**。

**コピー作業を減らしたい場合（非公式・高リスク）**: [`../../C1_cursor/1b_Cursorマニュアル/LINEチャット取得_非公式クライアントとリスク.md`](../../C1_cursor/1b_Cursorマニュアル/LINEチャット取得_非公式クライアントとリスク.md)
