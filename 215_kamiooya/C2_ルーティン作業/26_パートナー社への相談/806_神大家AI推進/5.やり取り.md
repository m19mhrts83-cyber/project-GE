**参加者**

**AI 活用提案の全体像**

3つの提案軸を紹介 ：

1. **成果報告の自動採点**：GAS（Google スプレッドシート用マクロ）と Gemini API を組み合わせた採点ツール
2. **コミュニティ Q&A チャットボット**：WeStudy のコメントを週次自動取得し、チャットで検索できる仕組み
3. **管理会社検索アプリ**：駅名入力で周辺管理会社一覧を表示

**成果報告 自動採点ツールのデモと説明**

- Google スプレッドシートに GAS スクリプトを組み込み、WeStudy から書き出した CSV を読み込んで Gemini API が採点
- 採点基準はスプレッドシート内のプロンプトに記載されており、変更可能
- デモ中にメニュー表示の不具合が発生。修正して再度共有予定
- **運用フロー**：CSV 差し替え → CSV 取り込み → 採点自動化実行 → AI 採点結果（J列）と手動採点結果（N列）を比較 → 差分を学習データとして補正 → 翌月以降の精度向上
- 初回の目標精度は 7〜8 割。100% は目指さず、人間の最終確認を前提とする
- **担当変更**：従来は林さんが採点を担当していたが、谷村さんへ引き継ぎ済み。谷村さんに試行してもらう予定
- Gemini の API 利用料は従量課金。試行期間は松野さんの API キーを使用し、本番移行時は切り替えを依頼
- 成果報告以外のトピック投稿への適用拡張も今後の検討事項

**コミュニティ Q&A チャットボットのデモと議論**

- WeStudy のコメントを週次スクレーピングで収集し、チャットで質問できるプロトタイプを構築済み
- 現状のデータソースはコミュニティ投稿のみ。動画（Vimeo に格納）の内容検索が次の課題
- **動画の文字起こし**：Notta を有料導入した経緯があるが、精度が低いとの報告あり。パワーポイントなどスライド資料の文字情報もデータベース化する案が出た
- 会員が知りたい情報は「講座動画の内容（どこで何を話しているか）」と「コミュニティ投稿」の2系統と整理
- 動画ファイルはすべて Vimeo に格納されており、WeStudy にはリンクが埋め込まれている構造
- 問い合わせ対応の自動返信（コメントへの AI 自動返信）へのニーズも挙がった。現状のチャットボットとは別の仕組みが必要

**管理会社検索アプリ**

- 駅名入力で周辺管理会社一覧を表示するウェブアプリを作成済み。三浦さんへデモ済み
- 既存の物件仲介会社一覧 Excel テンプレートとの競合・整合性について運営方針の確認が必要なため、木村さんへの相談を経て判断
- AI エージェントを活用した自動収集・定期アップデートも技術的には実現可能

**費用・契約・体制について**

- 松野さんの立場は個人法人での活動（所属会社とは無関係）
- **体制リスク**：松野さん1人に依存するリスクが指摘された。DX 除会（東海メンバー中心の5名グループ：千葉さん、伊山さん、岡田さん、本田さん等）が協力体制として機能する見込み
- AI によりソースコードの保守・引き継ぎが容易になっており、属人化リスクは以前より低減
- **報酬モデル**：月額顧問料＋成果報酬の組み合わせを検討。具体的な金額は未決定
- AI ツールの利用コストも含めた年間費用概算を松野さんに試算依頼

**リスクの整理**

- **運営内リスク**：AI に丸投げせず、最終判断は人間が行うマインドの維持が重要
