# Gmail → やり取り連携 セットアップ手順

やり取りフォルダにメール・添付を自動追記するために、順番に設定してください。

---

## Step 1: Google Cloud Console でプロジェクト作成

1. Google Cloud Console にアクセス  
   URL: **https://console.cloud.google.com/**  
   （リンクが開かない場合は、上記URLをブラウザのアドレスバーに直接コピー＆ペーストしてください）
2. 画面上部のプロジェクト選択 → **「新しいプロジェクト」** をクリック
3. プロジェクト名を入力（例: `yoritoori-gmail`）→ **作成**
4. 作成したプロジェクトが選択されていることを確認

---

## Step 2: Gmail API を有効化

1. 左メニュー **「API とサービス」** → **「ライブラリ」**
2. 検索欄で **「Gmail API」** を検索
3. **Gmail API** をクリック → **「有効にする」** をクリック

---

## Step 3: OAuth 認証情報の作成（credentials.json）

1. 左メニュー **「API とサービス」** → **「認証情報」**
2. **「認証情報を作成」** → **「OAuth クライアント ID」** を選択
3. **初回のみ**「同意画面の構成」が求められる場合:
   - **「同意画面を構成」** をクリック
   - ユーザータイプ: **「外部」** → 作成
   - アプリ名: 任意（例: `やり取り連携`）→ 保存
   - スコープ: 追加不要 → 保存
   - テストユーザー: 自分の Gmail アドレスを追加 → 保存
4. 再度 **「認証情報を作成」** → **「OAuth クライアント ID」**
5. アプリケーションの種類: **「デスクトップアプリ」**
6. 名前: 任意（例: `yoritoori-local`）→ **作成**
7. **credentials.json** がダウンロードされる
8. ダウンロードした `credentials.json` をこのフォルダ（`C1_cursor/1b_Cursorマニュアル/`）に保存する

---

## Step 4: 連絡先一覧にメールアドレスを登録

1. `C2_ルーティン作業/26_パートナー社への相談/000_共通/連絡先一覧.yaml` を開く
2. 各パートナー社の `emails: []` に、**実際のメールアドレス**を追加する  
   - 例: `emails: ["info@minitech.co.jp", "hayashi@minitech.co.jp"]`
3. `連絡先一覧.md` にも同じ内容を記入（人が見る用）
4. **重要**: 最低1社以上、1件以上のメールアドレスを登録しないとスクリプトは実行できない

---

## Step 5: 初回認可（token.json の作成）

1. ターミナルで以下を実行:
   ```bash
   cd "C1_cursor/1b_Cursorマニュアル"
   python gmail_to_yoritoori.py
   ```
2. 初回は認可用 URL が表示される → その URL を**ブラウザで開く**
3. 使用する **Google アカウント**を選択
4. 「このアプリは確認されていません」と出たら:
   - **「詳細」** をクリック
   - **「〇〇（安全ではないページ）に移動」** をクリック
5. **「アクセスを許可」** をクリック
6. 画面上に表示される **認証コード**（英数字）をコピー
7. ターミナルに戻り、「コード:」の後にコピーしたコードを貼り付けて Enter
8. 成功すると `token.json` が同じフォルダに作成される

---

## Step 6: 動作確認

1. パートナー社のメールアドレスから、**未読**のテストメールを1通用意する  
   （自分で自分に転送しても可）
2. 再度 `python gmail_to_yoritoori.py` を実行
3. 以下が確認できれば成功:
   - ターミナルに「追記: ○○ (メールアドレス) - 件名...」と表示される
   - 該当フォルダの `やり取り.md` に返信ブロックが追加されている
   - 添付があれば「添付」フォルダにファイルが保存されている

---

## トラブルシューティング

| 現象 | 対処 |
|------|------|
| credentials.json が見つかりません | Step 3 の保存先を確認。`gmail_to_yoritoori.py` と同じフォルダか |
| メールアドレスが1件も登録されていません | Step 4 で `emails` に少なくとも1件登録する |
| token の期限切れ | 再度 Step 5 を実行（token.json を削除してから実行しても OK）。同じ token を使う**いけともAIニュース（戸井さんニュース）**の GitHub Actions 用に、更新した token.json を Base64 にして **GMAIL_TOKEN_B64** も更新することを推奨（`Gmail token 更新時の連携（いけとも・パートナー）.md` 参照） |
| 未読メールがあるのに何も起きない | 送信元メールアドレスが連絡先一覧と完全一致しているか確認（大文字小文字は無視） |

## Gmail API の料金と制限

**Gmail API は無料です。** 使用量に応じた課金は発生しません。

### 制限（クォータ）

課金ではなく、**1分あたりの API 呼び出し数**に制限があります。超えると一時的にエラーになります。

| 制限種類 | 上限 | 超過時のエラー |
|---------|------|----------------|
| プロジェクト全体 | 1分あたり 1,200,000 クォータユニット | `rateLimitExceeded` |
| ユーザーあたり | 1ユーザーあたり1分あたり 15,000 クォータユニット | `userRateLimitExceeded` |

### 主な API のクォータ消費量（1回あたり）

| 操作 | クォータユニット |
|------|-----------------|
| メール送信 `messages.send` | 100 |
| メール取得 `messages.get` | 5 |
| メール一覧取得 `messages.list` | 5 |
| 添付ファイル取得 | 5 |

### 目安

- **ユーザーあたり 1分あたり 15,000 ユニット** → メール送信なら約 150 通/分、メール取得なら約 3,000 通/分まで可能
- パートナー社とのやり取りを自動化する程度の利用量であれば、制限に達することはほぼない

## Python の依存関係

初回のみ、以下のいずれかでインストール:
```bash
cd "C1_cursor/1b_Cursorマニュアル"

# 方法1: venv を使う（推奨）
python3 -m venv .venv_gmail
source .venv_gmail/bin/activate   # Windows: .venv_gmail\Scripts\activate
pip install -r requirements_gmail.txt

# 実行時
python gmail_to_yoritoori.py

# 方法2: 仮想環境を使わない場合
pip install --user -r requirements_gmail.txt
python gmail_to_yoritoori.py
```
