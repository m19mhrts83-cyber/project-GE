## 1. GitLens（eamodio.gitlens）

### 概要
Gitの履歴や変更情報をエディタ上で可視化する拡張機能。「誰が」「いつ」「何を」変更したかが一目で分かる。

### 主な機能
- **行ごとのBlame表示**: カーソルを置いた行に「誰がいつ変更したか」が薄く表示される
- **コミット履歴のグラフ表示**: サイドバーからコミットの流れを視覚的に確認
- **ファイル履歴**: ファイルの変更履歴を時系列で確認できる
- **変更の比較**: 過去のバージョンと現在のファイルを横並びで比較

### 使い方
1. **行のBlame確認**: ファイルを開くと、カーソル行の右側に薄いグレーで変更情報が表示される
2. **ファイル履歴を見る**: ファイルを右クリック → 「Open File History」
3. **サイドバーから操作**: 左サイドバーに追加される「GitLens」アイコンをクリック

### よく使うショートカット
| 操作 | キー |
|---|---|
| コミット詳細を表示 | 行のBlame表示をクリック |
| ファイル履歴を開く | 右クリック → Open File History |

### 設定（必要に応じて）
- Blame表示が邪魔な場合: 設定（`Cmd + ,`）で「gitlens.currentLine.enabled」を検索 → オフ
- 基本的にはデフォルトのままでOK

## 2. Markdown All in One（yzhang.markdown-all-in-one）

### 概要
Markdownファイルの編集を効率化する万能ツール。ショートカット、目次生成、テーブル整形など。

### 主な機能
- **プレビュー表示**: Markdownの見た目をリアルタイムで確認
- **目次（TOC）の自動生成**: 見出しから目次を自動作成
- **テーブル整形**: 表の列幅を自動で揃える
- **ショートカット**: 太字、斜体、リストなどをキー操作で

### よく使うショートカット
| 操作 | キー |
|---|---|
| プレビューを開く | `Cmd + Shift + V` |
| 横並びプレビュー | `Cmd + K` → `V` |
| **太字** | `Cmd + B` |
| *斜体* | `Cmd + I` |
| チェックリスト切替 | `Alt + C`（プレビュー上） |

### 目次を自動生成する方法
1. mdファイルを開く
2. `Cmd + Shift + P`（コマンドパレット）
3. 「Markdown All in One: Create Table of Contents」と入力
4. カーソル位置に目次が挿入される

### テーブルを整形する方法
1. 表（テーブル）を書いた状態で
2. `Alt + Shift + F`（フォーマット）
3. 列幅が自動で揃う

## 3. Markdown Preview Enhanced（shd101wyy.markdown-preview-enhanced）

### 概要
Markdown All in Oneより高機能なプレビュー。PDFエクスポートやmermaid図にも対応。

### 主な機能
- **高品質プレビュー**: よりきれいなレンダリング
- **PDFエクスポート**: Markdownから直接PDFを生成
- **HTMLエクスポート**: Webページとして出力
- **mermaid図対応**: フローチャートやシーケンス図をコード内に記述して表示

### 使い方
1. **プレビューを開く**: mdファイルを開いた状態で `Cmd + Shift + V`
2. **PDFにエクスポート**: プレビュー画面で右クリック → 「Chrome (Puppeteer) → PDF」
3. **HTMLにエクスポート**: プレビュー画面で右クリック → 「HTML → HTML (offline)」

### mermaid図の書き方（例）
mdファイル内に以下のように書くとフローチャートが表示される：

````markdown
```mermaid
graph TD
    A[ファイル編集] --> B[変更を確認]
    B --> C[ステージング]
    C --> D[コミット]
```
````

### 設定（必要に応じて）
- プレビューのテーマ変更: 設定で「markdown-preview-enhanced.previewTheme」を検索
- デフォルトのままでOK

## 4. indent-rainbow（oderwat.indent-rainbow）

### 概要
インデント（字下げ）のレベルごとに異なる色を付けて可視化する。

### 主な機能
- **インデントの色分け**: 各レベルのインデントが虹色で表示される
- **エラー検出**: インデントが不正な箇所が赤く表示される

### 使い方
- **特に操作不要**: インストールするだけで自動的に有効
- Pythonファイル（`.py`）を開くと効果が分かりやすい

### 表示例
```
def example():
│   if True:          ← 1段目（薄い色）
│   │   print("A")   ← 2段目（別の色）
│   │   if False:
│   │   │   pass      ← 3段目（さらに別の色）
```

### 設定（必要に応じて）
- 色が派手すぎる場合: 設定で「indentRainbow.colors」を検索してカスタマイズ可能
- 基本的にはデフォルトのままでOK

## 5. Better Comments（aaron-bond.better-comments）

### 概要
コード内のコメントを種類別に色分け表示する。TODO、注意、疑問点などが視覚的に区別できる。

### 主な機能
コメントの先頭に特定の記号を付けると色が変わる：

| 記号 | 色 | 用途 |
|---|---|---|
| `# !` | 赤 | 重要・警告 |
| `# ?` | 青 | 疑問・確認事項 |
| `# TODO` | オレンジ | やるべきこと |
| `# *` | 緑 | ハイライト・強調 |
| `# //` | 取り消し線 | 不要なコード |

### 使い方（Pythonの例）
```python
# ! これは重要な処理（赤で表示）
# ? この条件で合ってる？（青で表示）
# TODO エラーハンドリングを追加する（オレンジで表示）
# * この関数がメインの処理（緑で表示）

def send_mail():
    pass
```

### 設定（必要に応じて）
- 特に設定不要。インストールするだけで有効
- 色やタグをカスタマイズしたい場合: 設定で「better-comments.tags」を検索

## 6. Rainbow CSV（mechatroner.rainbow-csv）

### 概要
CSVファイルの各列を色分けして表示する。大量のデータでもどの列かが一目で分かる。

### 主な機能
- **列ごとの色分け**: CSVの各列が異なる色で表示される
- **列のホバー表示**: カーソルを置くとヘッダー名と列番号が表示される
- **RBQL（SQL風クエリ）**: CSVに対してSQLのような検索・フィルタが可能
- **列の整列表示**: CSVを表形式で見やすく表示

### 使い方
1. **色分け表示**: CSVファイルを開くだけで自動的に色分けされる
2. **列の整列**: `Cmd + Shift + P` → 「Rainbow CSV: Align」で列を揃えて表示
3. **SQLクエリ**: `Cmd + Shift + P` → 「Rainbow CSV: RBQL」でデータ検索

### RBQLの例（★資産計上_要確認.csv などで使える）
```sql
# 特定の列で絞り込み
SELECT * WHERE a1 == '建物'

# 金額でソート
SELECT * ORDER BY a5 DESC
```
※ a1 = 1列目、a2 = 2列目 ... と指定

### 設定（必要に応じて）
- CSVの区切り文字の変更: ステータスバー右下の「CSV」表示をクリック
- タブ区切り（TSV）にも対応

## 拡張機能の管理方法

### インストール済みの確認
- 左サイドバーの **拡張機能アイコン**（四角が4つのマーク）をクリック
- または `Cmd + Shift + X`

### 拡張機能の無効化・アンインストール
1. 拡張機能パネルを開く（`Cmd + Shift + X`）
2. 無効にしたい拡張機能をクリック
3. 「無効にする」または「アンインストール」をクリック

### 設定の変更
1. `Cmd + ,`（設定を開く）
2. 検索バーに拡張機能名やキーワードを入力
3. 該当の設定項目を変更

## 8. Excel Viewer（GrapeCity.gc-excelviewer）

### 概要
Cursor 上で Excel（.xlsx / .xls）や CSV をプレビュー・編集できる拡張機能。表形式のまま表示・編集可能。

### 主な機能
- **Excel プレビュー**: .xlsx / .xls を開くと表形式で表示される
- **編集**: セルの編集・保存・元に戻す・やり直しが可能
- **CSV 対応**: CSV も同様にプレビュー・編集できる

### 使い方
- **Excel ファイルをクリックするだけ**でプレビューが表示される
- エクスプローラーから .xlsx を開いても OK

### 補足（確認した内容）
- **Wijmo ライセンスのダイアログ**が出ることがある。「The Wijmo license in this application is invalid」と表示された場合は、**CLOSE で閉じてそのまま使ってよい**（プレビューは動作する）。気になる場合は「generate a 30-day license for free」で無料30日ライセンスを取得するか、別の拡張（Data Preview など）に乗り換え可能。
- **AI（Watson）は .xlsx の中身を直接読めない**（バイナリ形式のため）。内容を AI に扱ってもらう場合は、CSV にエクスポートするか、必要なセルをコピーしてチャットに貼る。

## 9. Docx Renderer（AdamRaichu.docx-viewer）※Word プレビュー用

### 概要
Word（.docx）を Cursor 内で読み取り専用プレビューできる拡張機能。軽量で、.docx を開くと文書として表示される。

### 主な機能
- **Word プレビュー**: .docx を開くと文書として表示される（読み取り専用）
- **docx-preview ベース**: 設定オプションで表示を調整可能

### 使い方
- **.docx ファイルをクリック**するとプレビューが表示される
- インストール: `Cmd + Shift + X` → 「Docx Renderer」または「Docx Viewer」で検索 → **Docx Renderer**（青い W アイコン、AdamRaichu）をインストール

### 補足（確認した内容）
- **ユーザー**は Cursor 上で Word の中身をプレビューして読める。
- **AI（Watson）は .docx の中身を直接読めない**（バイナリ形式のため）。拡張機能を入れても AI がファイルを「読む」ようにはならない。AI に内容を扱ってもらう場合は、必要な部分をコピーしてチャットに貼るか、Word から「名前を付けて保存」で .txt に書き出してそのファイルを参照させる。

### プレビューが開かない場合・設定の確認
- **現在 .docx の既定は「Office Viewer」**（`cweijan.officeViewer`）にしています。Docx Renderer は使わない場合は無効化してかまいません。
- **手動で開く**: エクスプローラーで .docx を**右クリック** → **「Open With...」** → **「Office Viewer」** を選ぶ。
- **設定の確認**: `Cmd + ,` → 検索で「editorAssociations」→ `*.docx` が `cweijan.officeViewer` になっているか確認。
- **Cursor の再読み込み**: `Cmd + Shift + P` → 「**Developer: Reload Window**」で再起動してから .docx を開き直す。

### Cursor 上で .docx プレビューがどうしても出ない場合（現実的な代替）
Cursor ではカスタムエディタの不具合で、拡張を替えても .docx プレビューが出ないことがあります。その場合は **Cursor の外で開く**のが確実です。
- **Finder から開く**: エクスプローラーで .docx を右クリック → **「Reveal in Finder」** → Finder でファイルをダブルクリック（Word や Pages で開く）。
- **ターミナルから開く**: `open "ファイルのフルパス"` で既定のアプリで開く。

### その他の候補（未導入）
- **Office Viewer**（cweijan.vscode-office）… Word・Excel・PDF をまとめてプレビュー。ライセンスダイアログの可能性あり。
- **Docx/ODT Viewer**（ShahilKumar.docxreader）… .docx 専用で高評価。アウトライン・検索・ズームなどが使える。

## トラブルシューティング

### 拡張機能が動かない場合
1. `Cmd + Shift + P` → 「Reload Window」でCursorを再読み込み
2. それでもダメなら、拡張機能を一度無効にして再度有効にする

### 動作が重くなった場合
1. 不要な拡張機能を無効化（アンインストールではなく無効化で試す）
2. GitLensの「行Blame表示」をオフにすると軽くなることがある
   - 設定 → 「gitlens.currentLine.enabled」→ オフ
