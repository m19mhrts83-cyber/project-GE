# LINE チャットを「コピーなし」で取り込む — 非公式クライアントについて

**運用上の優先**: まず **LINE 公式アプリのトーク履歴エクスポート／送信**で得た `.txt` を **`line_export_to_yoritoori.py`** で `5.やり取り.md` に載せる方法を推奨します（規約リスクが抑えめで、長文のドラッグコピーが不要になりやすい）。

個人／グループトークを **さらに API 的に** 取得するには、**非公式プロトコル実装**（いわゆる非公式クライアント）か **画面の自動操作** のどちらかになります。  
以下は **非公式クライアント側**の代表例と、試すときの前提・リスクです。

---

## いまのスクリプトとの役割分担

| 既にあるもの | できること |
|--------------|------------|
| `line_to_yoritoori_clip.py` | **すでにテキスト化された本文**を `5.やり取り.md` に整形追記する |
| `line_mac_open_chat_poc.py` | LINE を前面化し **検索欄に文字を流す**まで（履歴の取得・コピーはしない） |

非公式クライアントで「メッセージ一覧を取る」までできれば、その出力を **上記 clip スクリプトにパイプする**、または同じ見出し形式で追記する薄いラッパーを後から足せます。

## 非公式クライアントの代表（Python 系）

情報はリポジトリの更新状況で変わります。**試す直前に README・Issue を確認**してください。

### 1. CHRLINE 系（LINE の「Chrome / 内部 API」相当を叩くライブラリ）

- 元リポジトリ **DeachSword/CHRLINE** は **アーカイブ済み**（メンテ終了）の情報があります。
- コミュニティ側の **フォーク・パッチ版**（例: **CHRLINE-PatchV2** など）が、依存関係の修正目的で公開されていることがあります。
- 想定される流れ（概念のみ）: ログインセッションの確立 → トーク／メッセージ取得系の API を呼ぶ → テキスト化。

### 2. linepy 系（古い「unofficial messaging API」ラッパー）

- **linepy** および各種フォークは **メンテ状況・互換性にばらつき**があります。
- 過去には「利用でアカウント制限のリスクがある」旨が各所に書かれていることが多いです。

### 3. その他

- GitHub 上に断片的な実装や、別言語のクライアントが存在します。**スター数が少ない・最近更新がないものは、マルウェアやフィッシングの踏み台になっている例も業界一般でありうる**ため、**ソースを読める・信頼できる経路だけ**を使うことを強く推奨します。

## 許容しても残るリスク（要約）

1. **利用規約違反** … アカウント停止・利用制限の対象になりうる。  
2. **プロトコル変更** … LINE アプリ更新で突然動かなくなる（修復は自力）。  
3. **認証情報の取り扱い** … QR ログイン結果・トークン・デバイス鍵をスクリプトや `.env` に置くと、**漏洩＝LINE 乗っ取り級**。リポジトリに **絶対コミットしない**。  
4. **偽ライブラリ** … パッケージ名が似た悪意ある配布物に注意。  
5. **法的・契約上の問題** … 職場端末・管理物件のデータで使う場合は、別途判断が必要なことがあります。

**リスクを許容する場合でも**、可能なら **本番と別の LINE アカウントで試験**し、**専用の Python venv**・**作業用ディレクトリ**に閉じるのが無難です。

## 試験の進め方（おすすめ順）

### macOS（このパソコン）で CHRLINE を入れる

`git-repos/line_unofficial_poc/setup_phase_b_mac.sh` を実行する。  
`python-axolotl-curve25519` の **PyModuleDef m_size をパッチ**してからビルドするため、**素の `pip install CHRLINE` より通りやすい**。

```bash
cd ~/git-repos/line_unofficial_poc
chmod +x setup_phase_b_mac.sh activate_and_test.sh
./setup_phase_b_mac.sh
./activate_and_test.sh
```

### 一般的な手順

1. **専用フォルダ**に仮想環境を作る（上記 `line_unofficial_poc/.venv` でよい）。  
2. 選んだライブラリの **README のインストール手順どおり**に依存を入れる。  
3. **対話的にログイン**できるか、**1 ルームのメッセージが数件取れるか**だけを確認。  
4. 取れたテキストを、既存の  
   `line_to_yoritoori_clip.py --partner … --group --group-label …`  
   に **パイプ**するか、同じ Markdown ブロック形式で追記するスクリプトを追加する。

ここまでできたら、「プログラムでできたこと」は **明確に再現手順付きで言える**状態になります。

## 付録: 非公式ではないが「全文コピーを減らす」手段

LINE アプリの **トーク履歴のエクスポート／送信**（機種・版によりメニュー名が異なる）でファイルやメールにまとめる公式機能があります。  
エクスポート結果をパースする **linelog2py** 等のライブラリも存在します。  
これは **LINE 公式アプリの機能の範囲**で、非公式プロトコルは使いません。  
「規約リスクはさらに抑えたいが、手選択コピーは減らしたい」場合の中間案です。

- **認証トークンやログイン用の完成スクリプトは、このフォルダには置いていません**（漏洩・誤コミット防止のため）。
