# OneDrive 接続安定化のメモ

## 実施済み（前回・今回の検討）

1. **Python 実行環境のローカル化**
   - venv を OneDrive 上の `1b_Cursorマニュアル/.venv_gmail` ではなく、ローカルの `~/ProgramCode/venv_gmail` に作成。
   - スクリプト実行時のタイムアウトを回避。ルール（python-local-venv.mdc）で実行時は必ずローカル venv を指定。

2. **接続切れ時の振る舞いのルール化**
   - ファイル読み書きで ETIMEDOUT 等が出たら、リトライせず「接続が切れています」と伝える（onedrive-connection.mdc）。
   - 接続回復後に「接続した」と伝えてもらえば作業を再開。

## 追加で検討できる対策

### A. OneDrive アプリ・OS

- **OneDrive を最新版に更新**  
  Mac App Store または Microsoft 公式から。macOS 15 系では 15.2 以降で一部フリーズ問題が修正されている報告あり。
- **同期の「一時停止」→「再開」**  
  接続がおかしいと感じたら、OneDrive メニューで一度停止してから再開する。
- **重要なフォルダを「常にこのデバイスに保持」にする**  
  Cursor でよく開くフォルダ（例: 215 の `.cursor`、`C2_ルーティン作業`、`C1_cursor`）を Finder で右クリック → 「このデバイスで常に保持」を選ぶと、オンデマンドで都度取得する動きが減り、読み書きが安定しやすい。  
  ※ 容量に余裕がある場合。全フォルダを常に保持すると同期に時間がかかることがある。
- **macOS のバージョン**  
  可能なら macOS 15.2 以降にしておく（OneDrive の不具合修正のため）。

### B. ネットワーク

- **Wi‑Fi より有線（Ethernet）**  
  可能なら有線にすると、切断や遅延が減りやすい。
- **ルーターの再起動**  
  不安定な日は試す価値あり。
- **VPN**  
  業務で VPN を使っている場合は、切れるタイミングと一致していないか確認。必要なら一時オフにして OneDrive だけ試す。

### C. 作業のやり方で軽減

- **Cursor で開くワークスペースを絞る**  
  215 だけ開くなど、OneDrive のフォルダ数を減らすと、同時に触るファイルが減り負荷が下がりやすい。
- **編集は少ないファイルから**  
  接続が悪い日は、編集対象を少なめにして保存回数を減らす。
- **接続回復後に「接続した」と一言**  
  ルールに従い、エージェントが続きから再開できる。

### D. それでも厳しい場合の選択肢

- **よく触るフォルダだけローカルにコピーして作業し、あとで OneDrive に反映**  
  例: `git-repos` のように、重要なものはローカルで編集し、同期スクリプトで OneDrive と合わせる運用（すでに git-commit-habit で一部実施済み）。
- **.cursor/rules など小さく重要なフォルダ**  
  変更頻度が低いなら、ローカルにバックアップを取っておき、OneDrive が長期間使えないときだけローカルを参照する。
