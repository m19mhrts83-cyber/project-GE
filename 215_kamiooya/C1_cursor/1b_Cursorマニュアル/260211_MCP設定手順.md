# Cursor MCP 設定手順

Cursor で MCP（Model Context Protocol）サーバーを追加・設定する手順をまとめています。設定ファイルは `~/.cursor/mcp.json` です。

---

## 0. AI に依頼しながら進める：うまくいったやりとりの例

MCP のセットアップやエラー解消は、**Cursor の AI チャットに依頼しながら進める**とうまくいきやすいです。実際にうまくいった流れをまとめます。

### 依頼の流れ（実例）

| やりたいこと | 実際にしたこと・言い方 |
|--------------|------------------------|
| Gmail から情報を探したい | 「Gmail から〇〇を探してほしい」と依頼 → AI が「MCP が必要」と説明し、設定手順を案内 |
| Gmail MCP がエラーになっている | 「Gmail がエラーになっていました。エラー解消方法を確認してください」と依頼 |
| エラーの原因を特定したい | MCP 設定画面で「Error - Show Output」の内容を**共有** → AI がログから原因を特定（例：ポート競合） |
| 設定を修正してほしい | 「〇〇を修正して」と依頼 → AI が mcp.json を編集 |
| ツール数が上限を超えた | 「92ツール使っていて80以内に抑えたい。削りそうなツールを提案して」と依頼 → 候補リストを出してもらう |
| 手順をドキュメント化したい | 「MCP の設定手順を MD でまとめてほしい」と依頼 → 本ドキュメントを生成 |

### ポイント

1. **エラーが出たら「Show Output」を共有する**  
   AI はエラーログを見れば原因を特定しやすい。テキストやスクリーンショットで貼り付けるとよい。

2. **やりたいことと状況を一緒に伝える**  
   「Gmail からイベントの住所を探してほしい」→ まずは Gmail 連携の有無を確認する流れになる。

3. **設定の変更は AI に任せられる**  
   mcp.json の編集や、PORT の追加などは AI が実行できる。自分で書き換える必要はない。

4. **制約があれば先に伝える**  
   「80ツール以内に抑えたい」→ 削除候補の提案を依頼しやすい。

## 1. MCP とは

MCP により、Gmail・Notion・GitKraken などの外部サービスを Cursor から利用できます。各 MCP サーバーが「ツール」を提供し、AI がそれを呼び出して作業します。

---

## 2. 設定の開き方

1. **Cursor** を開く
2. **設定** を開く（`Cmd + ,` または メニュー → Cursor → Settings）
3. 左サイドバーで **「MCP」** を選択
4. 「Installed MCP Servers」で現在の MCP 一覧を確認

---

## 3. 現在の設定（2026年2月時点）

| MCP サーバー | 用途 | ツール数 |
|--------------|------|----------|
| **Tavily** | Web 検索 | 5 |
| **Notion** | Notion ワークスペースの検索・取得 | 13 + 1 リソース |
| **Gmail** | Gmail のメール検索・送信・管理 | 60+ |
| **GitKraken** | Git 操作・GitHub/Jira 連携 | 19 |

---

## 4. 設定ファイルの場所と編集

- **パス**: `~/.cursor/mcp.json`
- **編集**: Cursor の MCP 設定画面で編集するか、ターミナルで直接編集

```bash
# 設定ファイルを開く
open -e ~/.cursor/mcp.json
```

## 5. 各 MCP のセットアップ手順

### 5.1 Tavily（Web 検索）

1. [Tavily](https://tavily.com/) で API キーを取得
2. MCP 設定で「Tavily」を追加
3. 環境変数 `TAVILY_API_KEY` に API キーを設定

**主なツール**: 検索、URL 抽出、サイトクロール など

---

### 5.2 Notion

1. MCP 設定で「Notion」を追加（または `mcp-remote` で `https://mcp.notion.com/mcp` を指定）
2. 初回接続時に Notion の OAuth 認証が求められる
3. ブラウザで Notion にログインし、ワークスペースへのアクセスを許可

**主なツール**: 検索、ページ取得、ページ作成・更新、データベース操作 など

---

### 5.3 Gmail

#### 事前準備（Google Cloud Console）

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成 or 選択
2. **API とサービス** → **ライブラリ** で **Gmail API** を有効化
3. **認証情報** → **認証情報を作成** → **OAuth 2.0 クライアント ID**
4. アプリケーションの種類で **「デスクトップアプリ」** を選択
5. クライアント ID と クライアント シークレットを取得

#### リフレッシュトークンの取得

1. `mkdir -p ~/.gmail-mcp`
2. 上記の認証情報を `~/.gmail-mcp/gcp-oauth.keys.json` に保存（形式は Gmail_MCP_エラー解消手順.md 参照）
3. ターミナルで実行：

```bash
npx @shinzolabs/gmail-mcp auth
```

4. ブラウザで Google アカウントにログインし、表示された **REFRESH_TOKEN** をコピー

#### mcp.json への設定

Gmail の設定ブロックに以下を追加：

```json
"gmail": {
  "command": "/path/to/npx",
  "args": ["-y", "@shinzolabs/gmail-mcp"],
  "env": {
    "GMAIL_CLIENT_ID": "xxx.apps.googleusercontent.com",
    "GMAIL_CLIENT_SECRET": "GOCSPX-xxxx",
    "GMAIL_REFRESH_TOKEN": "1//0e...",
    "PORT": "3001"
  }
}
```

**PORT について**: ポート 3000 が他プロセスで使用されている場合は `PORT: "3001"` を追加（詳細は「Gmail_MCP_エラー解消手順.md」参照）。

---

### 5.4 GitKraken

- **GitLens** 拡張機能をインストールすると、GitKraken MCP が自動で利用可能になる
- 追加の設定は不要（GitHub/Jira などへの連携は各種サービスでログイン）

**主なツール**: git_add_or_commit, git_status, git_branch, git_push, issues_*, pull_request_* など

---

## 6. ツール数の制限（80ツール以内）

Cursor の Pro プランでは **合計 80 ツール以内** に収める必要があります。超える場合は、使わないツールを OFF にします。

- MCP 設定画面で各サーバーを開き、不要なツールのチェックを外す
- Gmail の削除候補例：S/MIME 関連、watch_mailbox、Delegates、転送設定、IMAP/POP など（詳細はチャット履歴参照）

---

## 7. エラーが出た場合

1. MCP 設定画面で **「Error - Show Output」** をクリックし、エラーメッセージを確認
2. **Gmail** のエラー → 「Gmail_MCP_エラー解消手順.md」を参照
3. **ポート競合**（EADDRINUSE: 3000）→ Gmail の env に `"PORT": "3001"` を追加

### 7.1 Notion：`notion-update-page` や `notion-create-pages` で「Expected object, received string」が出る場合

**現象**  
- ページのプロパティ更新で `notion-update-page` を呼ぶと `Expected object, received string (path: data)` となる。
- データベースに行を追加する `notion-create-pages` で `parent` を渡すと `Expected object, received string (path: parent)` となる。

**根本原因（Cursor クライアント側）**  
- **モデルが「Auto」のとき**、Cursor が MCP ツールのネストされたオブジェクト引数（`parent`、`data` など）を**二重に JSON 文字列化**して送信する。  
- Notion MCP サーバーはオブジェクトを期待しているが文字列を受け取るため、Zod 検証でエラーになる。  
- **モデルを Sonnet または Opus に明示選択**したときは正しくオブジェクトとして渡り、成功する。  
- 詳細は **「Notion_MCP_parent文字列化バグ_調査報告.md」** を参照。

**対策（即効）**  
1. **モデルを明示選択する**  
   チャット上部のモデル選択で **「Auto」ではなく「Sonnet」または「Opus」** を選び、その状態で Notion タスク登録を依頼する。  
2. **手動で Notion に登録する**  
   `Notionタスク_担当設定メモ.md` に登録手順と入力値を記載。データベース内で「新規」から行を追加する。  
3. **Notion API スクリプトを使う**  
   Integration トークンが用意できる場合、`create_notion_task.py` で直接 API を呼ぶ。手順は「Notion_タスク登録スクリプト.md」を参照。

**長期的な対応**  
- Cursor フォーラム（[#145807](https://forum.cursor.com/t/cursor-auto-selected-model-stringifies-mcp-tool-parameters/145807)）でエンジニアへ報告済み。修正リリースを待つ。
