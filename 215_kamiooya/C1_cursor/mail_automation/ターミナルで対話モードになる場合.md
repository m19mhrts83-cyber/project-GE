# ターミナルで Python 対話モード（>>>）になってしまう場合

## 原因

- **環境変数 `PYTHONINSPECT` が設定されている**  
  → Python の仕様では、非空の文字列（`1` だけでなく `0` も）で対話モードが有効になる。**`export PYTHONINSPECT=0` では逆効果。** 正しくは `unset PYTHONINSPECT`。
- **ターミナルの「デフォルトプロファイル」が Python になっている**  
  → シェルが bash/zsh ではなく Python として動いている。
- または **長いコマンド・パス内の記号** で引数が途切れ、`python` にスクリプト名が渡らず引数なし起動 → 対話モードになる。

## 対処A: AppleScript で実行する（.command が Python で開く場合の回避策）

**`送信_G2_260213.command` をダブルクリックすると Python 対話モードになる**場合、.command が Python で開かれている可能性があります。そのときは **AppleScript** を使ってください。

1. **スクリプトエディタ**（アプリケーション → ユーティリティ）を開く
2. **ファイル → 開く** で `mail_automation` フォルダ内の **送信_G2_260213.scpt** を開く
3. 上部の **▶（実行）** ボタンをクリック
4. **ターミナル**が開き、送信処理が始まる
5. ブラウザが開いたら Gmail 認証を完了する

## 対処B: Mac のターミナルで直接実行する

1. **ターミナル**（アプリケーション → ユーティリティ）を開く
2. 次のコマンドをコピーして貼り付け、Enter を押す:

```bash
bash "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/mail_automation/送信_G2_260213.sh"
```

3. ブラウザが開いたら Gmail 認証を完了する

※ Cursor のターミナルではなく、**Mac 標準のターミナル**を使うこと。

## 対処C: .command をダブルクリック（通常時）

**送信_G2_260213.command** を Finder でダブルクリックすると、Mac の「ターミナル」が開いて実行されます。対話モードになる場合は対処AまたはBを使用してください。

## 対処0: スクリプトをそのまま実行する（修正済み）

`送信_G2_260213.sh` に **unset PYTHONINSPECT** を追加済みです（`PYTHONINSPECT=0` は逆効果）。次のコマンドで対話モードにならずに実行できるはずです。

```bash
bash "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/mail_automation/送信_G2_260213.sh"
```

まだ `>>>` になる場合は、手動で環境変数を消してから実行してください。

```bash
unset PYTHONINSPECT
bash "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/mail_automation/送信_G2_260213.sh"
```

## 対処1: シェルを「zsh」にしてから実行する（Cursor のターミナル）

1. Cursor で **新しいターミナル** を開く（＋ または `` Ctrl+Shift+` ``）。
2. ターミナル右側の **▼** をクリック → **「デフォルトのプロファイルを選択」** または **「プロファイルを選択」** で **「zsh」** を選ぶ。
3. もう一度、**新しいターミナル** を開き直す（これで zsh が使われる）。
4. 次を **1行だけ** コピー＆ペーストして実行する（スクリプト経由なので引用符で途切れない）:

```bash
bash "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/mail_automation/送信_G2_260213.sh"
```

これで `send_mail.py` が引数付きで実行され、対話モードには入りません。

## 対処2: Mac の「ターミナル」アプリで実行する

- Cursor ではなく **Mac 標準の「ターミナル」** を開く。
- 通常は最初から zsh なので、そのまま次を実行:

```bash
bash "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/mail_automation/送信_G2_260213.sh"
```

## ポイント

- **長いコマンドを直接貼り付けない** → パス内のカンマや括弧で引数が壊れ、`python` が「スクリプトなし」で起動することがある。
- **`bash 送信_G2_260213.sh` のようにスクリプト経由で実行する** → 引用符はスクリプト内で正しく書かれているので、対話モードにならない。

## 再起動したあとにやること（Cursor でずっと >>> になる場合）

1. **Cursor を終了してから起動し直す**
2. **新しいターミナルを1つ開く**（既存のタブは使わない）
3. ターミナル枠の**右側の ▼** をクリック → **「zsh」** または **「bash」** を選ぶ（**「Python」は選ばない**）
   - タイトルに「Python」と出ているターミナルは、入力が Python に渡っているので、そのタブは使わない
4. プロンプトが **`$`** や **`%`** になっていることを確認してから、次を実行:

```bash
bash "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/mail_automation/送信_G2_260213.sh"
```

**それでも同じなら** → Cursor のターミナルは使わず、**Mac の「ターミナル」アプリ**を開き、上と同じコマンドを貼り付けて実行する（こちらは通常 zsh なので対話モードになりにくい）。

## すでに >>> が出ているとき

- いったん Python を終了する: **`exit()`** と打って Enter。
- その後、上記のとおり **プロファイルを zsh に変える** か **Mac のターミナル** で、`bash ...送信_G2_260213.sh` を実行する。
