# メール送信が届かないときの調査メモ

## 今回の状況（2/13送信）

- **現象**: 送信コマンド実行後、Gmailの「送信済み」にメールが増えていない
- **ログ**: `logs/send_history.log` に **2月13日の記録が1件もない**
  - 最後の成功は 2026-02-07（15件送信）
  - 2/13に送信処理が完了していない、またはログまで到達していない

## 想定されるつまずきポイント

### 1. Gmail認証で止まっている（最も有力）

- スクリプトは送信前に **Gmail API の OAuth** を行う
- トークンが無い／期限切れのときは `run_local_server(port=0)` で **ブラウザが開き、認証完了を待つ**
- Cursor の「バックグラウンド実行」で動かした場合:
  - ブラウザは開いても、**認証後のコールバック（localhost）がプロセスに届いていない**可能性
  - またはブラウザが開いていない／気づかず閉じた
- **結果**: 認証待ちのままタイムアウトし、1通も送信されず終了（その場合もログに残らないことがある）

### 2. トークン期限切れ

- `token.pickle` は期限があり、切れると再認証（ブラウザ）が必要
- 再認証も「1」と同様に、**フォアグラウンドで実行しないと完了しにくい**

### 3. 過去ログで出ていた別のエラー（参考）

- **Invalid To header**: 宛先メールアドレスの形式不備（Excelのセルに余分な文字や改行が入っている等）
- **メールアドレスの列が見つかりません**: シートの1行目（ヘッダー）に「メール」「mail」「email」などを含む列がない
- **メールアドレスが1件も見つかりませんでした**: 2行目以降に `@` を含むセルがない

## 推奨する確認手順

### Step 1: フォアグラウンドで実行して挙動を確認する

**Cursor のターミナル** または **Mac の「ターミナル」アプリ** で、以下を **バックグラウンドにせず** そのまま実行する:

```bash
cd "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/mail_automation"
./venv/bin/python send_mail.py \
  --md-file "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C2_ルーティン作業/24_空室対策メール履歴/260213_G2_空室対策.md" \
  --excel-file "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/20_【空室対策】【修繕】【売却】/21_【空室対策】募集,ステージング,物件管理/★管理会社一覧.xlsx" \
  --sheet-name "G2" \
  --yes
```

- **ブラウザが開いたら** → Googleアカウントで認証を完了する
- **ターミナルに表示されるメッセージ**（「○件送信しました」「エラー: …」など）をそのまま確認する
- エラーが出た場合は、その全文を控える

### Step 2: 送信しないドライランで MD/Excel を確認する

送信せずに「件名・本文・宛先」だけ確認する:

```bash
cd "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/mail_automation"
./venv/bin/python send_mail.py \
  --md-file "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C2_ルーティン作業/24_空室対策メール履歴/260213_G2_空室対策.md" \
  --excel-file "/Users/matsunomasaharu/Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/20_【空室対策】【修繕】【売却】/21_【空室対策】募集,ステージング,物件管理/★管理会社一覧.xlsx" \

```

- ここでエラーになる場合 → ファイルパス・シート名・Excelの列（メール/会社/担当）のどれかが原因
- ここまで成功する場合 → 問題はほぼ Gmail 認証か送信処理

### Step 3: ログの再確認

送信実行後、次のファイルに1行追加されているか確認する:

- `C1_cursor/mail_automation/logs/send_history.log`

- 日時・件名・送信先件数・SUCCESS/FAILED が書かれていれば、スクリプトは最後まで動いている
- 2/13の行が依然として無い場合は、認証や送信の手前で止まっているか、異常終了している

## まとめ

- **送信済みにメールが増えない＋2/13のログがない** ことから、**Gmail認証の待ちで止まっている**か、**認証が完了しないまま終了している**可能性が高いです。
- 対処としては **「バックグラウンドにせず、ターミナルでフォアグラウンド実行し、ブラウザ認証を完了させる」** ことをおすすめします。
- 上記 Step 1 を実行したときのターミナル出力（またはエラー全文）があれば、さらに原因を絞り込めます。
