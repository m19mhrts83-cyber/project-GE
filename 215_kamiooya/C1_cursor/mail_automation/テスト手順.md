# メール送信システム テスト手順

このドキュメントでは、メール自動送信システムを初めて使用する際の安全なテスト手順を説明します。

## ⚠️ 重要: 必ずテストを実施してください

本番のメールを送信する前に、必ずテスト送信を行ってください。これにより以下を確認できます：

- システムが正しく動作すること
- メールの内容が意図通りであること
- 送信先の設定が正しいこと
- Gmail APIの認証が正常に機能すること

1. 新しいExcelファイルを開く

#### サンプルデータ


**重要**: 必ず自分のメールアドレスを使用してください。

### ステップ4: テスト用MDファイルを作成

簡単なテストメールの内容を作成します。

```markdown

お世話になっております。

これはメール自動送信システムのテスト送信です。

このメールが正常に受信できていれば、システムは正常に動作しています。

本メールへの返信は不要です。

よろしくお願いいたします。
```

### ステップ5: Dry-run モードでプレビュー

まず、実際に送信せずにプレビューを確認します。

## 🔍 トラブルシューティング

### エラー: `credentials.json not found`

**原因**: Gmail APIの認証情報ファイルがありません。

**解決方法**: `README_Gmail_API_Setup.md` を参照して、`credentials.json` を作成・配置してください。

### エラー: `openpyxlがインストールされていません`

**原因**: 依存パッケージがインストールされていません。

**解決方法**:
```bash
pip3 install -r requirements.txt
```

### エラー: `Excelファイルが見つかりません`

**原因**: ファイルパスが間違っているか、ファイルが存在しません。

**解決方法**: 
- ファイルパスを確認
- 相対パスの場合は、現在のディレクトリを確認（`pwd`コマンド）

### エラー: `メールアドレスの列が見つかりません`

**原因**: メールアドレス列を自動検出できませんでした。

**解決方法**:
```bash
--email-column "メールアドレス"
```
で列名を明示的に指定してください。

### 認証画面が表示されない

**原因**: ブラウザのポップアップブロック

**解決方法**: 
- ターミナルに表示されるURLをコピーして手動でブラウザで開く
- ポップアップブロックを解除

### メールが届かない

**確認事項**:
1. 迷惑メールフォルダを確認
2. Gmailの「送信済み」フォルダを確認（自分から自分への送信の場合）
3. メールアドレスが正しいか確認
4. スクリプトの実行ログでエラーがないか確認

## 🎓 ベストプラクティス

### テスト送信のタイミング

以下の場合は、必ずテスト送信を行ってください：

1. 初めてシステムを使用する時
2. 長期間使用していなかった後
3. MDファイルの形式を大きく変更した時
4. 新しいExcelファイルを使用する時
5. Pythonスクリプトを更新した時

### 段階的な送信

大量の送信先がある場合：

1. まず自分宛にテスト送信（1件）
2. 次に少数の実際の送信先にテスト（5〜10件）
3. 問題なければ全体に送信

### 送信制限の管理

- Gmail APIの制限: 個人アカウント 500通/日
- 送信前に必要な送信件数を確認
- 大量送信の場合は複数日に分散

## ❓ よくある質問

### Q1: テスト送信は何回やってもいいですか？

**A**: はい。ただし、Gmail APIの送信制限（500通/日）には注意してください。テストメールも1通としてカウントされます。

### Q2: テストファイルは削除していいですか？

**A**: システムの動作確認が完了すれば、テスト用のMDファイルとExcelファイルは削除しても構いません。ただし、将来のトラブルシューティングのために保管しておくことを推奨します。

### Q3: 本番環境でもDry-runを使えますか？

**A**: はい。本番送信の前に `--dry-run` オプションを使って、送信内容を再確認することを推奨します。

### Q4: 認証は毎回必要ですか？

**A**: いいえ。初回のみです。認証後は `token.pickle` が作成され、次回以降は自動的に認証されます。
