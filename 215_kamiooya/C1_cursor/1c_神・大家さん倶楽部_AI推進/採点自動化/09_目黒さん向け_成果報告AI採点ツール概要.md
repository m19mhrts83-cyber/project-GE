# 神大家さんクラブ向け WeStudy成果報告AI採点ツール 概要

このドキュメントは、WeStudy の成果報告コミュニティ投稿をもとに、
Google スプレッドシート＋Google Apps Script（GAS）＋ Gemini API を使って
「自動採点」と「根拠の一覧出力」を行うツールの概要と、利用時の注意点をまとめたものです。

---

## 1. 全体像（何ができるか）

### 目的

- WeStudy の「成果報告コミュニティ」の投稿について、
  - あらかじめ定義した「得点基準表」に沿って
  - AI（Gemini）に点数を付けさせ、
  - 点数＋採点根拠（本文中のフレーズ）をスプレッドシートに一覧で出す。
- 月次集計（MVP 選定 など）の事前作業を自動化し、人手は「最終確認と微調整」に集中できる状態を目指します。

### 処理の流れ（ざっくり）

1. 管理画面から成果報告コミュニティの CSV を出力し、Google Drive にアップロード
2. スプレッドシートのメニューから CSV を `元データ` シートに取り込み
3. スプレッドシートのメニュー「採点実行」で、1 件ずつ Gemini に投げて採点
4. 結果を `採点結果` シートに追記（点数、ルールID、根拠フレーズなど）
5. 必要に応じて、人が点数を補正し、その差分を `補正学習データ` に蓄積

### 入出力のイメージ

| 項目 | 内容 |
|------|------|
| 入力 | WeStudy CSV（コメントID、本文、投稿者名、親コメントID など） |
| 中間 | スプレッドシート＋GAS（ルール表＋プロンプト＋Gemini API） |
| 出力 | スプレッドシートの `採点結果` シート（採点結果の一覧） |

## 3. 目黒さん側で「必ず変更が必要な箇所」

ここを **神大家さんクラブ側の環境に合わせて差し替えないと、松野個人のアカウントや Drive に依存したまま** になってしまいます。

### 3-1. Gemini API キー（GEMINI_API_KEY）

**現状**

- 検証のため、松野個人の Gemini API キーを `設定` シートに一時的に入れて動作確認をしました。
- このまま運用すると、**松野個人のクレジットが消費されてしまう**ため、本番運用では使えません。

**必要な対応**

- 神大家さんクラブ側で、以下のどちらかで **専用の API キー** を取得していただきたいです。
  - 目黒さんご自身の Google アカウントで取得したキー
  - 神大家さんクラブ用の共通アカウントで取得したキー
- 取得したキーを、スプレッドシートの `設定` シート内の `GEMINI_API_KEY` セルに貼り付けてください。

**ポイント**

- API キーは「文字列 1 行」をそのまま貼るだけです。
- キーを入れ替えたら、`MAX_ROWS_PER_RUN` を小さめ（3〜5 件）にして一度テスト実行し、エラーにならないこと・`採点結果` に行が追加されることを確認してください。

### 3-2. Drive 上の CSV ファイル ID（DRIVE_CSV_FILE_ID）

**現状**

- 松野側の Drive に置いた検証用 CSV の File ID を設定しています。

**必要な対応**

1. westudy 管理画面から、成果報告コミュニティの CSV を出力。
2. 神大家さんクラブ指定の Google Drive フォルダに CSV をアップロード。
3. Drive 上でその CSV を開き、URL から File ID を取得（`/file/d/XXXXXXXXXXXXXXXXXXXX/view` の `XXXXXXXXXXXXXXXXXXXX` 部分）。
4. スプレッドシートの `設定` シート内の `DRIVE_CSV_FILE_ID` に、その ID を貼り付け。

**運用上のおすすめ**

- CSV の **ファイル名を固定**し、毎月の CSV を同名ファイルで「上書きする」運用にすると、File ID を毎月変えずに済みます（Drive の「上書きアップロード」機能を使うイメージ）。

### 3-3. 実行パラメータ（任意調整）

`設定` シートで調整できる主なパラメータ：

| 項目 | 説明 |
|------|------|
| `MAX_ROWS_PER_RUN` | 1 回の「採点実行」で何件まで処理するか。最初は 3〜5 件でテスト → 問題なければ 50 件、100 件などに拡大する運用がおすすめ。 |
| `INCLUDE_REPLIES` | TRUE: 返信コメントも採点対象に含める／FALSE: 親コメント（スレッドの頭）だけを採点 |
| `採点バージョン` | 得点ルールやプロンプトを大きく変えたときに番号を変えておくと、「どのバージョンのロジックで採点されたか」を後から比較するときに便利。 |

## 4. Google Cloud 側の前提条件（クォータまわり）

AI 採点を安定稼働させるために、**Google Cloud 側の設定が通っていること**が前提になります。

### 必須条件

- 対象プロジェクトに **課金アカウントが紐づいている**（クレジットカード登録など）。
- Gemini API（`generativelanguage.googleapis.com`）が有効化されている。
- `https://ai.dev/rate-limit` で該当プロジェクトの
  - `GenerateRequestsPerDayPerProjectPerModel`
  - `GenerateContentInputTokensPerModelPerMinute`
  などの limit が **0 より大きくなっている**。

### もし limit がすべて 0 の場合

- どれだけ `runScoring` を実行しても、**すべて 429（RESOURCE_EXHAUSTED）で弾かれ、採点件数 0 のまま**です。
- 課金アカウントの有効化や利用枠の反映が終わるまで待つ必要があります。

## 5. 月次運用フロー（想定）

1. **CSV 出力**
   - WeStudy の管理画面から成果報告コミュニティの CSV を出力する。
   - 対象期間（例: 当月 1〜末日）を指定。

2. **Google Drive に CSV をアップロード**
   - 神大家さんクラブ指定のフォルダにアップロード。
   - ファイル名を固定して上書き運用にすると、設定変更が少なくて済みます。

3. **スプレッドシートの設定確認**
   - `設定` シートで、
     - `GEMINI_API_KEY` が神大家さんクラブのものになっていること
     - `DRIVE_CSV_FILE_ID` が今月の CSV を指していること
     - `MAX_ROWS_PER_RUN` の値（テスト時は 3〜5、慣れてきたら拡大）
   を確認する。

4. **CSV 取り込み**
   - メニューから「CSV取込（Drive File ID）」を実行し、`元データ` に反映。

5. **AI 採点の実行**
   - メニューから「採点実行」を実行。
   - ログシートに「採点件数: n」と出ていること、`採点結果` に行が追加されていることを確認。

6. **結果確認と補正**
   - `採点結果` シートを見ながら、人が必要に応じて点数を補正（`手動補正点` 列）。
   - 「補正学習データに追加」メニューを実行し、`補正学習データ` シートに差分を蓄積。

7. **MVP などの集計に利用**
   - `集計` シートや別ファイルで、`採点結果` を参照してランキングや候補抽出を行う。

## 6. 今後の拡張の方向性（メモ）

- **westudy API が整備された場合**
  - 今回は CSV 前提ですが、API が用意されたら、「CSV出力 → Driveアップロード」の部分を「GAS から westudy API を直接呼んで JSON を取得」に置き換えることを想定しています。

- **集計・可視化**
  - MVP 候補の自動ピックアップ
  - 分野別の得点傾向の可視化
  - メンバーごとの成果推移ダッシュボード など

まずは「月次で CSV を入れて採点できる」状態まで動かし、運用の中で見えてきた課題をもとに、少しずつ手を入れていくイメージです。
