# 採点が 0 件のときの確認リスト

「採点実行」で **採点件数: 0** とログに出た場合、以下の順でスプレッドシートと GAS の設定を確認してください。

---

## 1. 元データシートに「採点対象の行」があるか

- **シート名**が **`元データ`** になっているか。
- **2行目以降**に、WeStudy の CSV から取り込んだデータ（コメントID・本文・投稿者名など）が入っているか。
- ヘッダーだけでデータ行が 0 行だと、採点対象がなく **採点件数: 0** になります。
- **「CSV取込（Drive File ID）」** で CSV を読み込んだあと、もう一度「採点実行」を試してください。

---

## 2. 設定シートの値（MAX_ROWS_PER_RUN など）

- **`設定`** シートに **1回の実行で採点する最大件数**（例：`MAX_ROWS_PER_RUN` や「採点上限」など）を書いている場合、**0** になっていないか確認してください。
- 0 のままだと 1 件も処理されず、採点件数: 0 になります。テストなら **5** や **10** などに変更して再実行してください。

---

## 3. すでに採点済みの行だけを飛ばす仕様の場合

- スクリプトが「**採点結果シートにすでに存在するコメントIDはスキップする**」動きになっている場合、**元データの全行がすでに採点済み**だと 0 件になります。
- **採点結果** シートに、今回の元データと同じコメントIDがすでに並んでいないか確認してください。
- 初回実行や、**別の CSV（別月など）** を取込した直後なら、採点結果が空か、該当コメントIDが無い状態で実行してください。

---

## 4. 得点基準シート

- **`得点基準`** シートが存在し、**ヘッダーと 1 行以上のルール** が入っているか確認してください。
- 得点基準が空だとプロンプトが作れず、処理が行われない（または 0 件扱い）になる実装の可能性があります。

---

## 5. GAS のスクリプト プロパティ（API キー）

- **拡張機能 → Apps Script** でプロジェクトを開き、**プロジェクトの設定 → スクリプト プロパティ** に **`GEMINI_API_KEY`** が設定されているか確認してください。
- キーが空や誤りだと、API 呼び出しでエラーになり、**エラーログは出るが採点件数は 0** のような動きになることがあります。
- **実行ログ**（Apps Script の「実行数」や「ログ」）に、429 や 403 などの API エラーが出ていないかもあわせて確認してください。

---

## 6. シート名・列の並び

- マニュアルや `01_シート設計.md` で想定している **シート名**（`元データ`・`得点基準`・`採点結果`・`設定`・`ログ`）と、実際のスプレッドシートが一致しているか確認してください。
- **元データ** の **列の並び**（例：1列目＝コメントID、本文の列など）が、GAS のコードで参照している列番号とずれていないか確認してください。ずれていると「有効な行」が 0 件と判定されることがあります。

---

## 7. ログシートでエラー内容を確認

- **ログ** シート（例：gid=435404304 のタブ）に、**採点件数: 0** 以外のメッセージ（エラー文やスタックトレース）が出力されていないか確認してください。
- 同じ実行で **「Exception」「Error」「429」「403」** などが出ていれば、API キー・課金・クォータが原因の可能性があります。  
  → その場合は `11_Google_AI_StudioでAPIキーを取得する手順（課金で止まったとき）.md` や、Google Cloud の課金有効化を再確認してください。

---

## 確認の優先順位（短く）

| 順番 | 確認すること |
|------|----------------|
| 1 | **元データ** に 2 行目以降のデータがあるか |
| 2 | **設定** の MAX_ROWS_PER_RUN（または採点上限）が 0 になっていないか |
| 3 | **採点結果** に同じコメントIDがすでに並んでおらず、未採点の行があるか |
| 4 | **得点基準** に 1 行以上のルールがあるか |
| 5 | **GEMINI_API_KEY** がスクリプト プロパティに正しく入っているか |
| 6 | **ログ** に API エラー（429 など）が出ていないか |

---

上記を確認しても 0 件のままの場合は、**ログシートの該当実行の前後のログ** と、**元データ・設定・採点結果の各シートの先頭数行**（個人情報を除く）をメモし、サポートや開発者に共有すると原因を特定しやすくなります。
