# Markdown ファイルが読み込めなくなったときの確認

`04_gas_導入手順.md` など、OneDrive 上の Markdown が「突然読み込めない」場合に考えられる原因と対処です。

---

## 1. OneDrive の「オンラインのみ」になっている

**原因**  
ファイルが **オンラインのみ**（クラウド上だけにあり、PC にダウンロードされていない）のとき、アプリや Cursor が開こうとするとダウンロード待ちでタイムアウトしたり、開けないことがあります。

**対応**  
- エクスプローラー（Finder）でそのファイルを **右クリック** → **「常にこのデバイスに保持する」**（Always keep on this device）を選ぶ。  
- または、ファイルを開いて中身を一度表示させ、同期が終わるのを待ってから再度開く。

---

## 2. OneDrive の同期エラー・競合

**原因**  
別デバイスで編集した・競合が起きた・同期が途中で止まったなどで、ファイルが「利用できない」状態になることがあります。

**対応**  
- **OneDrive のアイコン（タスクトレイなど）** を開き、同期エラーや保留中の項目がないか確認する。  
- エラーが出ているファイルは、OneDrive の Web（onedrive.live.com）で開いて内容を確認し、必要なら別名で保存し直す。  
- 問題のファイルを **別のフォルダにコピー** してから、コピー先を開いてみる（コピーで同期状態がリセットされることがある）。

---

## 3. ファイルの文字コード・改行

**原因**  
UTF-8 以外の文字コードや、特殊な改行コードで保存されていると、一部のエディタやツールで「読み込めない」「文字化け」になることがあります。

**対応**  
- メモ帳や別エディタで開き、**「名前を付けて保存」** で **UTF-8**（BOM なし推奨）で上書き保存する。  
- 改行コードを **LF** に統一してみる。

---

## 4. ファイルの破損

**原因**  
保存の失敗や同期の不具合で、ファイルの中身が壊れている可能性があります。

**対応**  
- OneDrive の **バージョン履歴** を開き、**以前のバージョン** を復元する。  
  （ファイルを右クリック → バージョン履歴 / Version history）  
- 別のバックアップ（他の PC、別フォルダのコピー）があれば、そこから内容を復元する。

---

## 5. パスが長い・特殊文字

**原因**  
フォルダ階層が深い、またはファイル名に特殊な文字が含まれると、一部の環境で開けなくなることがあります。

**対応**  
- ファイルを **フォルダ階層の浅い場所**（例：ドキュメント直下）にコピーして開いてみる。  
- ファイル名を **半角英数字中心** に変えてみる（例：`04_gas_導入手順.md` → `04_gas_intro.md`）。

---

## 6. Cursor / エディタ側のキャッシュ

**原因**  
エディタのキャッシュやインデックスがおかしくなり、特定ファイルだけ開けないことがあります。

**対応**  
- Cursor を **再起動** する。  
- **「ファイルを開く」** で該当の .md を直接指定して開く。  
- ワークスペースをいったん閉じて、フォルダを開き直す。

---

## まとめ（確認の順番）

| 順番 | 確認・対応 |
|------|-------------|
| 1 | OneDrive でそのファイルを **「常にこのデバイスに保持する」** にする |
| 2 | OneDrive の同期エラー・保留がないか確認し、問題があれば Web で開いて別名保存やコピーを試す |
| 3 | **バージョン履歴** で過去の版に戻す |
| 4 | 別のアプリ（メモ帳・別エディタ）で開けるか試し、UTF-8 で保存し直す |
| 5 | Cursor の再起動、または「ファイルを開く」で直接指定して開く |
