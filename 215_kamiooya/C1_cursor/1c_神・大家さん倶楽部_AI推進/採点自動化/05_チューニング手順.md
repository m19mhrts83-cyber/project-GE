# テスト・チューニング手順

## 1. 検証データ作成（人手採点データ）

以下を実行して検証用CSVを作る。

```bash
python3 05_generate_calibration_csv.py
```

出力:

- `05_calibration_samples.csv`（991件）

このCSVには `human_score`（人手得点）と投稿本文が含まれる。

---

## 2. 初期検証（サンプル50件）

1. `05_calibration_samples.csv` から50件を抽出して `元データ` に投入
2. `runScoring` 実行
3. `採点結果` の `得点` と `human_score` を比較

推奨指標:

- 一致率: `AI得点 == 人手得点`
- 誤差平均: `ABS(AI得点 - 人手得点)` の平均
- 大誤差率: `ABS差 >= 3` の割合

---

## 3. ルール調整方針

ズレが多い場合、次の順に修正する。

1. `02_得点基準_seed.csv` の `判定基準` を具体化
2. `03_prompt_spec.md` の対象外判定条件を厳密化
3. `MAX_ROWS_PER_RUN` を下げて再実行（API安定化）

---

## 4. 代表的なズレパターン

- **返信コメントの誤採点**: `親コメントID` がある短文を0点に寄せる
- **融資情報 vs 成果報告の誤分類**: 分類キーワードを明記する
- **高得点の過剰付与**: 「再現手順の具体性」を必須条件に加える

---

## 5. 本番運用時のガード

- `採点結果` の `手動補正点` を初期3か月は必須運用
- 月次で20件だけ抜き取りレビューし、基準表を更新
- 基準更新時は `採点バージョン` を `v1 -> v2` のように更新
