# WeStudy採点自動化 シート設計

## 1. 目的

WeStudyの成果報告CSVを取り込み、配点基準に基づいてGeminiで採点し、月次集計まで実行するためのスプレッドシート構成。

---

## 2. シート一覧

1. `設定`
2. `元データ`
3. `得点基準`
4. `採点結果`
5. `集計`
6. `ログ`

---

## 3. 各シート定義

## `設定`

| 列A | 列B | 用途 |
|---|---|---|
| GEMINI_API_KEY | (APIキー) | Gemini APIキー |
| GEMINI_MODEL | gemini-2.0-flash | 利用モデル |
| TARGET_MONTH | 2026-03 | 集計対象月 (YYYY-MM) |
| DRIVE_CSV_FILE_ID | (任意) | Drive上CSVのファイルID |
| SOURCE_SHEET_NAME | 元データ | 元データシート名 |
| RULES_SHEET_NAME | 得点基準 | 得点基準シート名 |
| RESULT_SHEET_NAME | 採点結果 | 結果シート名 |
| LOG_SHEET_NAME | ログ | ログシート名 |
| MAX_ROWS_PER_RUN | 50 | 1回の実行で採点する最大件数 |
| INCLUDE_REPLIES | FALSE | 返信コメントを採点対象に含めるか |

---

## `元データ`

CSV取り込み先。列構成はWeStudy CSV準拠。

| 列名 | 説明 |
|---|---|
| コメントID | 一意キー |
| 投稿日時 | 投稿日時 |
| 投稿者名 | 会員番号＋氏名を想定 |
| 投稿者メール | 投稿者メール |
| コメント内容 | 本文 |
| 親コメントID | 返信の場合に値あり |
| IP アドレス | 監査用途 |
| ユーザーエージェント | 監査用途 |
| 取込日時 | GASで追加 (任意) |

---

## `得点基準`

Geminiに渡す評価ルールの正本。

| 列名 | 説明 |
|---|---|
| ルールID | 例: 購入AP1 |
| 大分類 | 成果報告系_購入 / 成果報告系_他 / 情報共有系 |
| 中分類 | 物件購入_AP / 空室対策 / 融資情報 など |
| レベル | 1,2,3 |
| 観点 | 採点観点の要約 |
| 判定基準 | Geminiに読ませる詳細基準 |
| 点数 | このルールの点数 |
| キーワード補助 | 機械判定補助キーワード（任意） |
| 有効 | TRUE/FALSE |

---

## `採点結果`

1行=1投稿の評価結果。

| 列名 | 説明 |
|---|---|
| コメントID | 元データと突合 |
| 投稿日時 | 元データ |
| 投稿者名 | 元データ |
| 親コメントID | 元データ |
| コメント内容 | 元データ |
| 対象判定 | 成果報告対象/対象外 |
| 推定分野 | 物件購入/空室対策/修繕/融資/情報共有/その他 |
| ルールID | Geminiが選んだルール |
| 得点 | Gemini採点 |
| 根拠 | Gemini説明 |
| 信頼度 | high / medium / low |
| 手動補正点 | 人手で補正した点数 |
| 最終点 | `=IF(ISBLANK(手動補正点),得点,手動補正点)` |
| 採点バージョン | ルール改定用 |
| 採点日時 | 実行日時 |
| エラー | APIエラー時の内容 |

---

## `集計`

`採点結果` を月次集計する。

| 指標 | 内容 |
|---|---|
| 投稿者別合計点 | 最終点の合計 |
| 投稿数 | 投稿件数 |
| 平均点 | 合計点/投稿数 |
| 分野別内訳 | 分野ごとの点数 |
| ランキング | 合計点順 |

---

## `ログ`

処理ログ用。

| 列名 | 説明 |
|---|---|
| 時刻 | ログ時刻 |
| レベル | INFO/WARN/ERROR |
| 関数名 | 実行関数 |
| コメントID | 対象行 |
| メッセージ | 詳細 |

---

## 4. 運用フロー

1. `元データ` にCSVを取り込む（手動貼り付け or Drive読込）。
2. `得点基準` を最新化する。
3. `runScoring` を実行し `採点結果` を生成。
4. `集計` で月次ランキングを確認。
5. 必要に応じて `手動補正点` を入力し最終確定。
