# 登録承認メール通知（Phase 0）— GAS + MailApp

新規登録時に管理者へ「承認をお願いします」メールを送るための送信口。  
月額0円。パートナー用 Gmail API（OAuth）とは別経路（Apps Script の `MailApp`）。

## 構成

| 役割 | 場所 |
|------|------|
| GAS ソース（貼り付け用） | [`gas/registration_notify/Code.gs`](gas/registration_notify/Code.gs) |
| テスト送信 | [`scripts/test_registration_notify.py`](scripts/test_registration_notify.py) |
| 秘密・URL（正本） | `~/git-repos/.env.jarvis_private` の `NOTIFY_*` |

設定3点（移管時に差し替える統一ルール）:

1. `NOTIFY_WEBHOOK_URL` — GAS ウェブアプリ URL  
2. `NOTIFY_SHARED_SECRET` — 共有シークレット  
3. `NOTIFY_ADMIN_TO` — 承認依頼の宛先（GAS の `ADMIN_TO` と揃える）

---

## Phase 0 — あなたが行う手順（約10〜15分）

### 1. Apps Script プロジェクトを作成

1. [script.google.com](https://script.google.com/) を開く（倶楽部用意の個人用 Google アカウント推奨）  
2. 「新しいプロジェクト」  
3. プロジェクト名: `kamiooya-qa-registration-notify`  
4. エディタのコードをすべて消し、リポジトリの `gas/registration_notify/Code.gs` を貼り付けて保存  

### 2. スクリプトプロパティを設定

「プロジェクトの設定」（歯車）→「スクリプト プロパティ」→「プロパティを追加」:

| プロパティ | 値 |
|------------|-----|
| `SHARED_SECRET` | `.env.jarvis_private` の `NOTIFY_SHARED_SECRET` と同じ文字列 |
| `ADMIN_TO` | 管理者の受信メール（`.env` の `NOTIFY_ADMIN_TO` と同じ）。**複数宛先はカンマ区切り**（下記） |
| `APP_URL` | （任意）`https://ma-54t2keqdelz3.raimo-app.buzz/` |

#### 複数の管理者へ送るとき（カンマ区切り）

コード改修は不要。**プロパティは `ADMIN_TO` の1つだけ**を使い、値をカンマ区切りにする。

1. Apps Script → プロジェクトの設定 → スクリプト プロパティの **`ADMIN_TO`** を編集  
2. 値の例:

```text
admin1@example.com, admin2@example.com
```

3. 同じ文字列を `.env.jarvis_private` の **`NOTIFY_ADMIN_TO`** にも書く（運用メモ上の正本と揃える）  
4. 再デプロイは不要（プロパティ変更だけで次回送信から反映）  
5. `scripts/test_registration_notify.py` または `scripts/test_raimo_notify_registration.py` で1通送り、両方の受信箱を確認  

注意:

- `NotifyAdmin2` のような **別プロパティ名は使わない**（現行 GAS は `ADMIN_TO` しか読まない）  
- アドレスの間は `,`（カンマ）。余分な空白はあってもよい  
- 宛先を増やすほど Gmail の日次送信上限に近づく（承認通知程度なら通常問題なし）

### 3. ウェブアプリとしてデプロイ

1. 「デプロイ」→「新しいデプロイ」  
2. 種類の横「設定」→「ウェブアプリ」  
3. 説明: `phase0 registration notify`  
4. 実行ユーザー: **自分**  
5. アクセスできるユーザー: **全員**  
6. 「デプロイ」→ 初回は権限を承認  
7. **ウェブアプリの URL** をコピー  

### 4. Jarvis 側に URL を保存

`.env.jarvis_private` の次を更新:

```bash
NOTIFY_WEBHOOK_URL=（コピーしたウェブアプリURL）
```

（`NOTIFY_SHARED_SECRET` / `NOTIFY_ADMIN_TO` は Jarvis が Phase 0 着手時に雛形投入済みならそのまま）

保存後、チャットで「URLを保存した」と一声ください。Jarvis がテスト送信を実行します。

### 5. テスト送信（Jarvis または手元）

```bash
cd ~/git-repos/215_kamiooya/C1_cursor/1c_神・大家さん倶楽部_AI推進/神・大家さん倶楽部情報Q&Aチャットボット
/Users/matsunomasaharu2/selenium_env/venv/bin/python scripts/test_registration_notify.py
```

管理者の受信箱に件名「【神大家Q&A】新規登録の承認をお願いします」が届けば Phase 0 完了。

## 運営移管チェックリスト

### A. GAS（Google Drive）

1. 運営アカウントの Drive へ本プロジェクトをコピー／移管  
2. スクリプトプロパティ（`SHARED_SECRET` / `ADMIN_TO` / `APP_URL`）を運営側で再設定  
3. **運営アカウントでウェブアプリを新規デプロイ**（実行: 運営 / アクセス: 全員）  
4. 新しい Webhook URL を控える  
5. テスト1通で、送信元が運営 Gmail になることを確認  
6. 旧（個人）デプロイを停止

## Phase 2 — 新規登録へ接続（完了手順）

登録成功後にフロントから `POST /notify/registration` を呼ぶ。  
**通知が失敗しても登録は成功のまま**（`app.js` の `notifyRegistrationPending` がエラーを握りつぶす）。

| 役割 | 場所 |
|------|------|
| 接続箇所 | `app.js` → `handleRegister` 成功直後 |
| 公開 | `scripts/publish_raimo_1346.py` |

確認: アプリで新規登録 → 管理者宛に承認依頼メールが届く。  
DB に承認待ちユーザーが増えていれば登録成功。メールだけ来なくてもユーザーは残る。

## Phase 3 — 承認完了メール（申請者宛て）

管理者が「承認」を押したあと、**申請者のメール**へ承認完了通知を送る。  
アクセス用 URL は GAS の **`APP_URL`**（`.env` の `NOTIFY_APP_URL` と揃える）。

| 役割 | 場所 |
|------|------|
| GAS | `gas/registration_notify/Code.gs`（`type=approval`） |
| API | `POST /notify/approval` |
| 接続 | `app.js` → `confirmApproveUser` 成功直後の `notifyApprovalCompleted` |

### GAS の更新（必須・あなた側）

1. [script.google.com](https://script.google.com/) で `kamiooya-qa-registration-notify` を開く  
2. エディタのコードを、リポジトリ最新の `Code.gs` で**全置換**して保存  
3. スクリプトプロパティで **`APP_URL`** がアプリ URL になっているか確認（未設定なら追加。`NOTIFY_APP_URL` と同じ）  
4. 「デプロイ」→「デプロイを管理」→ 鉛筆で**新バージョン**、または「新しいデプロイ」  
5. Webhook URL が変わったら `.env.jarvis_private` の `NOTIFY_WEBHOOK_URL` を更新し、「保存した」と一声  

### テスト

```bash
cd ~/git-repos/215_kamiooya/C1_cursor/1c_神・大家さん倶楽部_AI推進/神・大家さん倶楽部情報Q&Aチャットボット

# Raimo 公開後
/Users/matsunomasaharu2/selenium_env/venv/bin/python scripts/publish_raimo_1346.py
```

確認: 管理画面で承認待ちを承認 → 申請者宛に件名「【神大家Q&A】登録が承認されました」が届く。  
**メール失敗でも承認（status=approved）は成功のまま。**

## Phase 4 — パスワード再設定メール

承認済みユーザーが「パスワードを忘れた方」からメールで再設定 URL を受け取り、URL から新パスワードだけを設定する。

| 役割 | 場所 |
|------|------|
| GAS | `gas/registration_notify/Code.gs`（`type=password_reset`） |
| API | `POST /auth/reset-password/check` · `POST /auth/forgot-password/issue` · `POST /auth/forgot-password/validate` · `POST /auth/reset-password`（**token 必須**） · `POST /notify/password-reset` |
| UI | `index.html` / `app.js`（hash `#reset-password?token=`） |
| DB | `users.reset_token` / `users.reset_expires_at`（有効期限は日本時間で発行日＋7日の **23:59** まで。メール表記も同形式。期限判定はフロント） |

### 画面メッセージ

| 入力メール | 表示 |
|------------|------|
| 未登録 | 「登録されたメールアドレスを入力してください」 |
| 承認待ち | 「まだ管理者の承認待ちです。承認後にパスワードを再設定できます」 |
| 承認済み | 「再設定用のURLをメールで送信しました」（再設定画面へは遷移しない） |

### GAS の更新（必須・あなた側）

1. [script.google.com](https://script.google.com/) で `kamiooya-qa-registration-notify` を開く  
2. エディタをリポジトリ最新 `Code.gs` で**全置換**して保存  
3. 「デプロイ」→「デプロイを管理」→ 鉛筆で**新バージョン**  
4. URL が変わったら `.env.jarvis_private` の `NOTIFY_WEBHOOK_URL` を更新し、「保存した」と一声  

### テスト

```bash
cd ~/git-repos/215_kamiooya/C1_cursor/1c_神・大家さん倶楽部_AI推進/神・大家さん倶楽部情報Q&Aチャットボット
/Users/matsunomasaharu2/selenium_env/venv/bin/python scripts/test_registration_notify.py --password-reset
/Users/matsunomasaharu2/selenium_env/venv/bin/python scripts/publish_raimo_1346.py
```

確認: アプリで再設定メール送信 → 受信箱の URL を開く → 新パスワード設定 → ログイン。  
**メール失敗でも token 発行は成功のまま**（届かないときは再送。運用上はもう一度「再設定メールを送る」）。
