# 東海労金 振込画面遷移 試行履歴

振込画面へ進めない事象について、実施した対応と結果を記録し、原因切り分けができるようにする。

---

## 課題の整理

| 項目 | 内容 |
|------|------|
| 現象 | ログイン後、トップの「この口座から」の「振込」から次の画面に進めない |
| **次の画面** | **振込クリック直後の画面**（日付振込選択などが出る、そのひとつ先の画面） |
| 目標 | 振込画面まで自動で遷移し、振込先・金額などを自動入力したい |
| 備考 | **手動で「振込」をクリックしても次の画面に進めない**との報告あり（自動化だけでなく画面遷移そのものの可能性） |

## 重要な切り分け情報（試行 #6 以降）

- **手動で「振込」をクリックしても次の画面に進めない**  
  → 自動化の不具合ではなく、次のいずれかの可能性がある。
  - セッション・認証が次の画面に引き継がれていない
  - リンクが POST や別ウィンドウで、期待したURLに遷移していない
  - サイト側のエラー・制限（時間切れ、同一画面へのリダイレクトなど）
  - 「次の画面」の認識の違い（例: BPT001Dispatch のさらに先の画面を指している）

## 現在の設定（config_tokairokin.yaml の該当部分）

```yaml
# 振込画面へ直接URLで遷移（優先）
transfer_direct_path: "BPT001Dispatch"
# 未指定時のみ「振込」ボタンクリックを試行
transfer_menu_button_selector: "#btn002-1"
# 日付振込（オプション・XPath はコメントアウト）
# transfer_dated_transfer_button_selector: "..."
```

- **優先**: `transfer_direct_path` が指定されているため、ログイン後に現在URLのパスだけ BPT001Dispatch に置き換えて直接遷移する。
- 振込ボタンクリック（#btn002-1 / テキスト / iframe / JS クリック）は、`transfer_direct_path` 未指定時のみ実行される。

## 次の切り分け・確認案

1. **手動で「振込」をクリックしたとき**
   - アドレスバーのURLは変わるか（例: BPT001Dispatch になるか）。
   - エラーメッセージや「アクセスできません」等の表示が出ていないか。

2. **直接URL（BPT001Dispatch）をアドレスバーに貼って Enter したとき**
   - 同じログイン済みブラウザで、`https://www.parasol.anser.ne.jp/ib/BPT001Dispatch?ServerID=...&B=2972&PT=BS` を開いて次画面が表示されるか。

3. **「次の画面」の定義**
   - → **振込クリック直後の画面**（ユーザー確認済み）

4. **別ブラウザ・シークレット**
   - 別ブラウザやシークレットでログインし直し、手動で「振込」をクリックした場合に同じ現象かどうか。

---

### 1. どこまで完了しているか

| 項目 | 状態 |
|------|------|
| ログイン〜再ログイン | 完了（合言葉・OTP は手動入力＋Enter で次へ） |
| 振込画面への遷移ブロック | **未到達**（Enter を押すとここから実行される） |
| 試行ログの書き出し | 振込遷移ブロックの**最後**で実行する実装済み。未実行のためログ未生成 |
| 試行履歴への「直近実行結果」追記 | 上記ブロック完了時に実行する実装済み。未実行のため未追記 |
| 振込先・金額の自動入力 | 振込画面に遷移した**後**の処理のため、まだ実行されていない |

---

### 2. 何ができていないか

| 項目 | 内容 |
|------|------|
| 振込遷移ブロックの実行 | 合言葉／OTP の Enter を押さないと開始されない。Enter を押したうえで、まだログが1回も出ていない＝**ブロック完了に至った実行がまだない** |
| 振込クリック直後の画面表示 | 手動で「振込」を押しても進まない報告あり。**画面・セッション側の要因の可能性**が高い |
| 直近実行結果の可視化 | ログ未生成のため「どの試行が success/failed か」が試行履歴に残っていない |

---

### 3. 対策

| 対策 | 内容 |
|------|------|
| **A. ログを1回出す** | スクリプト実行後、**合言葉・OTP を入力したら必ずターミナルで Enter を押す**。すると振込遷移ブロック（transfer_direct_path → セレクタ → iframe → メイン → 手動案内 or キーワード）が実行され、最後に `東海労金_振込_実行ログ_YYYYMMDD_HHMMSS.txt` が作成され、試行履歴に「直近実行結果」が追記される。 |
| **B. 直接URLの妥当性確認** | ログイン後のトップで、アドレスバーの URL をコピーし、`/ib/XXXXDispatch` の XXXX を `BPT001Dispatch` に置き換えたURLを同じタブで開く。振込クリック直後の画面が出るか確認する。出ない場合は ServerID やパラメータが不足している可能性がある。 |
| **C. 手動クリック時のURL確認** | 手動で「振込」をクリックした直後に、アドレスバーがどう変わるか（BPT001Dispatch になるか、エラー画面か、変わらないか）をメモする。試行履歴の「次の切り分け・確認案」に追記すると原因切り分けに使える。 |
| **D. ログをより手前で出す（任意）** | 「ログイン処理が完了しました」の直後や、合言葉 Enter の直後にログの一行だけ書き出すようにすると、**振込遷移ブロックに到達する前**でも「どこまで進んだか」をファイルで確認できる。 |

## 振り返り：振込プロセスの分析（2026-03-06 実行結果）

### 1. ターミナル実行結果から分かったこと

| 順番 | 処理 | 結果・備考 |
|------|------|------------|
| 1 | ログイン〜再ログイン | 完了 |
| 2 | 合言葉の一時停止 → Enter | 次へ進む |
| 3 | **振込画面への遷移** | **2パターンあり**（下記） |
| 4 | 振込先指定〜銀行・支店・口座・金額・実行画面へ | **自動入力で成功**（手動クリック後に実行された回） |
| 5 | ワンタイムパスワード（OTP）一時停止 | 手動入力後 Enter |
| 6 | ブラウザは手動で閉じる案内 → Enter | スクリプト終了 |

**振込画面への遷移の2パターン（ログより）**

- **A. transfer_direct_path（BPT001Dispatch）が success のとき**  
  ログイン直後の現在URLを BPT001Dispatch に置き換えて直接遷移。このときは「振込先を指定」以降の自動入力まで進んでいる（23:13, 23:14 の実行）。
- **B. transfer_direct_path が failed（URLが変化しなかった）のとき**  
  セレクタ・iframe・メインの「振込」クリックも失敗し、**手動で「振込」をクリック → Enter** で次へ。その後に「振込先を指定」〜実行画面へが自動で実行されている（23:22, 23:33 の実行）。

**ユーザー報告との対応**

- 「振込ボタンを押して Enter のあと、振込時間の表示やよく分からない内容がたくさん出て、時間が経ってから銀行名・支店コードが入力できる状態になった」  
  → **振込画面（BPT001Dispatch）表示直後は、振込時間の案内などが先に表示され、銀行・支店入力欄が使えるまでラグがある**と解釈できる。
- 「銀行口座の入力画面に行くまでにかなり時間がかかっている」  
  → 上記の「表示が切り替わるまでの待ち」が不足している可能性。
- 「振込画面の操作が手動でないといけない」  
  → 振込**ボタン**（この口座から→振込）は、direct_path が失敗する回は手動クリックが必要。**振込先・金額の入力**は、正しい画面に遷移できていれば自動入力で成功している。

### 2. 原因の整理

| 原因 | 内容 |
|------|------|
| 銀行入力まで時間がかかる | 振込画面表示後、**「振込先を指定」や銀行コード入力欄が使えるまで**の待ちが `wait_after_page`（2秒）のみで、実態より短い。振込時間の案内などが消えてフォームが表示されるまで、明示的に待っていない。 |
| 振込ボタンが手動になることがある | `transfer_direct_path` が「URLが変化しなかった」で失敗するケースがある（ログイン直後の current_url が `/ib/` を含まない、または別形式の可能性）。その場合はセレクタ・iframe・メインのクリックも失敗し、手動クリック＋Enter にフォールバックしている。 |

### 3. 実施した対策

- **振込画面表示後、銀行入力が使えるまで待つ**  
  「振込先を指定」ボタン（または銀行コード入力欄）が **表示・クリック可能になるまで最大 N 秒待つ**（WebDriverWait）を追加。`wait_after_page` だけに頼らず、要素の出現を待ってから「振込先を指定」をクリックする。
- **待機時間の設定**  
  `config_tokairokin.yaml` に `wait_for_transfer_form_ready`（秒）を追加。デフォルト 30 秒など、環境に合わせて変更可能にする。
- **振込ボタンの自動化**  
  direct_path が失敗する原因（ログイン直後のURL形式）の切り分けは別途必要。現状は「手動クリック → Enter で以降は自動」の流れを維持しつつ、**フォームが使えるまでの待ち**を入れることで、手動クリック後の「時間がかかる」「よく分からない表示」のあいだでも、自動入力が確実に動くようにする。
