# 東海労金 振込フォーム自動入力 設計

## 目標

- **入力**: 振込先（銀行・支店・口座番号）・振込金額をユーザーが指定
- **自動**: ログイン → 合言葉 → 振込画面 → **フォーム入力** → 確認画面まで
- **手動**: ワンタイムパスワード（OTP）の入力と最終実行のみユーザーが行う

---

## 振込に必要な情報

| 項目 | 例（三菱UFJ 熱田支店） | 形式 |
|------|------------------------|------|
| 銀行コード | 0005 | 4桁 |
| 支店コード | 405 | 3桁 |
| 口座種別 | 3（普通） | 1桁 |
| 口座番号 | 0526519 | 7桁 |
| 振込金額 | 100000 | 円 |

※銀行によっては「銀行名・支店名」で検索選択する形式もある

---

## 実装方針

### 1. パラメータの受け渡し

**方法A: コマンドライン引数**（推奨）
```
python fetch_after_login.py tokairokin --transfer \
  --bank 0005 --branch 405 --account 0526519 --amount 100000
```

**方法B: 環境変数**
```
TOKAIROKIN_TRANSFER_BANK=0005
TOKAIROKIN_TRANSFER_BRANCH=405
TOKAIROKIN_TRANSFER_ACCOUNT=0526519
TOKAIROKIN_TRANSFER_AMOUNT=100000
```

**方法C: config の transfer セクション**（登録済み振込先を再利用する場合）

### 2. フォーム構造の調査が必要

東海労金の振込画面は parasol.anser.ne.jp 上。実際の HTML を確認して以下を特定する必要がある：

- 銀行コード入力欄のセレクタ（id/name）
- 支店コード入力欄
- 口座番号入力欄
- 金額入力欄
- 「確認」ボタン

**調査方法**: 振込画面を開いた状態でブラウザの開発者ツール（F12）で要素を検証

### 3. 処理フロー

```
1. ログイン（既存）
2. 合言葉（既存・自動）
3. 再ログイン（既存・自動）
4. 振込メニュークリック（既存）
5. 【新規】振込フォーム入力
   - 銀行・支店・口座・金額を入力
   - 「確認」クリック
6. 【新規】OTP入力のため一時停止
   - 「ワンタイムパスワードを入力してください」と表示
   - ユーザーが手動で OTP 入力 → 実行
7. keep_browser_open で終了
```

### 4. フォームセレクタの設定例（要実機確認）

```yaml
# config_tokairokin.yaml に追加
transfer_form:
  bank_code_selector: "#txtBox020"      # 例
  branch_code_selector: "#txtBox021"
  account_type_selector: "select[name='accountType']"  # 普通=3
  account_number_selector: "#txtBox022"
  amount_selector: "#txtBox023"
  confirm_button_text: "確認"
```

1. **振込画面を開き、開発者ツール（F12）で各入力欄を検証**
2. **config_tokairokin.yaml の `transfer_form` にセレクタを設定**

---

## 注意事項

- 振込は金銭操作のため、実行前に必ず内容をユーザーに確認させる
- フォーム構造はサイト改修で変わる可能性があるため、セレクタは config で上書き可能に
- 登録済み振込先から選択する形式の場合は、別のロジック（リストから選択）が必要
