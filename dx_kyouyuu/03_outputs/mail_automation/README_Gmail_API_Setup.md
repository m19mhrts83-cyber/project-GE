# Gmail API セットアップガイド

このガイドでは、メール自動送信スクリプトを使用するために必要なGmail APIの設定方法を説明します。

## 前提条件

- Googleアカウント（個人用Gmail）
- インターネット接続
- ブラウザ

## セットアップ手順

### 1. Google Cloud Consoleでプロジェクトを作成

1. [Google Cloud Console](https://console.cloud.google.com/)にアクセス
2. ログインしていない場合は、Googleアカウントでログイン
3. 画面上部の「プロジェクトを選択」→「新しいプロジェクト」をクリック
4. プロジェクト名を入力（例：`Mail Automation`）
5. 「作成」をクリック

### 2. Gmail APIを有効化

1. 作成したプロジェクトを選択
2. 左側のメニューから「APIとサービス」→「ライブラリ」を選択
3. 検索バーに「Gmail API」と入力
4. 「Gmail API」をクリック
5. 「有効にする」ボタンをクリック

### 3. OAuth同意画面を設定

#### 【重要】「スコープ」セクションが表示されない場合について

2024年6月時点で、Google Cloud Consoleの仕様が変更され、「OAuth同意画面」の設定中に「スコープ」セクションが表示されないケースがあります。

**手順：**

1. 左側のメニューから「APIとサービス」→「OAuth同意画面」を選択
2. ユーザータイプで「外部」を選択し、「作成」をクリック
3. 必須項目を入力：
   - **アプリ名**: `Mail Automation`（任意の名前）
   - **ユーザーサポートメール**: 自分のメールアドレス
   - **デベロッパーの連絡先情報**: 自分のメールアドレス
4. 「保存して次へ」をクリック

> **この時、「スコープ」設定画面が表示されず、「指標」や「プロジェクトの診断」など他の画面が出る場合、そのまま「保存して次へ」を繰り返して「OAuth同意画面」の設定を最後まで完了させてください。**

**スコープの追加は、OAuth同意画面作成後に行っても問題ありません。**

#### スコープ追加方法（後から設定する場合）

1. 左側メニューで再度「APIとサービス」→「OAuth同意画面」を選択します。
2. 画面中ほどの「スコープ」セクションに「編集」ボタンがあるのでクリックします。
3. 「スコープを追加または削除」を押します。
4. 検索バーに `gmail.send` と入力し、  
   `https://www.googleapis.com/auth/gmail.send` にチェックを入れて「更新」や「追加」を押します。
5. 追加内容を確認し、「保存」してください。

- スコープ設定画面が見つからない場合や迷った場合は、Google Cloud Consoleの別タブで「認証情報 ＞ 対象のクライアントID詳細」から「OAuth同意画面」へアクセスし、スコープ編集メニューを探してください。

**このように、スコープは「OAuth同意画面」設定後でも自由に追加・編集が可能です。必ず `https://www.googleapis.com/auth/gmail.send` が含まれていることを確認してください。**

6. 「テストユーザー」の画面が表示されない場合、そのまま「認証情報」作成に進んで問題ありません。
7. 「OAuthクライアントID作成」ボタンを押して次のステップへ進んでください。

> ※スコープ設定画面で誤っても、ウィンドウ右上の「キャンセル」で戻り、再度やり直し可能です。


### 4. OAuth 2.0 クライアントIDを作成

1. 左側のメニューから「APIとサービス」→「認証情報」を選択
2. 画面上部の「+認証情報を作成」→「OAuth クライアント ID」をクリック
3. アプリケーションの種類で「デスクトップアプリ」を選択
4. 名前を入力（例：`Mail Automation Client`）
5. 「作成」をクリック
6. 作成されたクライアントIDの画面が表示されるので、「JSONをダウンロード」をクリックします。

> **もし「JSONをダウンロード」をクリックしてもファイルがダウンロードできない場合：**
>
> - ブラウザのポップアップブロックが有効になっている場合は解除してください。
> - 画面をリロードして再度試してみてください。
> - クライアントシークレット（OAuth 2.0 クライアントID）の詳細画面にダウンロードマーク（↓）が表示されている場合、そのアイコンをクリックすればJSONファイルをダウンロードできます。これで問題ありません。
> - もし画面を閉じてしまった場合や再度ダウンロードしたい場合は、  
>   左側メニュー「APIとサービス」→「認証情報」から該当する「OAuth 2.0 クライアントID」を探し、その行をクリックしてください。詳細画面が開いたら、ダウンロードマークまたは画面内の「ダウンロード」ボタンからJSONファイルを取得できます。
7. ダウンロードしたJSONファイルを `credentials.json` にリネーム
8. このファイルを以下の場所に配置：
   ```
   C1_cursor/mail_automation/credentials.json
   ```

### 5. 初回認証

以下の手順で認証を行います。

1. ターミナルまたはコマンドプロンプトを開きます。
2. このディレクトリ（`C1_cursor/mail_automation/`など）に移動します。
3. `python main.py`（または使用するスクリプト名）を実行してください。
4. スクリプトを初めて実行すると、自動的にブラウザが開き、Googleの認証画面が表示されます。

※ ブラウザが自動で開かない場合は、ターミナルに表示されるURLをコピーして手動でブラウザに貼り付けてください。

1. Googleアカウントを選択
2. 「アプリは Google で確認されていません」という警告が表示される場合：
   - 「詳細」をクリック
   - 「Mail Automation（安全ではないページ）に移動」をクリック
3. 「このアプリは確認されていません」と表示されますが、自分で作成したアプリなので問題ありません
4. 権限の確認画面で「許可」をクリック
5. 認証が完了すると `token.pickle` ファイルが自動生成されます

## セキュリティに関する注意事項

### 重要なファイル

以下のファイルは機密情報を含むため、厳重に管理してください：

- `credentials.json`: OAuth 2.0クライアントの認証情報
- `token.pickle`: アクセストークン（自動生成）

### .gitignoreへの追加

Gitを使用している場合は、これらのファイルを必ず`.gitignore`に追加してください：

```
# Gmail API認証情報
C1_cursor/mail_automation/credentials.json
C1_cursor/mail_automation/token.pickle
```

## トラブルシューティング

### エラー: `credentials.json not found`

- `credentials.json`ファイルが正しい場所に配置されているか確認してください
- ファイル名が正確に `credentials.json` になっているか確認してください

### エラー: `The user has not granted the app permission`

- OAuth同意画面で正しいスコープ（`gmail.send`）が設定されているか確認してください
- `token.pickle`を削除して、再度認証を試してください

### 認証画面が表示されない

- ブラウザのポップアップブロックを確認してください
- ターミナルに表示されるURLを手動でコピーしてブラウザで開いてください

### 送信制限について

Gmail APIには以下の制限があります：

- **個人アカウント**: 1日500通まで
- **Google Workspace**: 1日2,000通まで

制限を超えた場合は、翌日まで待つ必要があります。
