## 2. Cursor（カーソル）の紹介と基本設定

- **Cursor**は、AIと対話しながらファイル・コードを編集できるツール。フォルダ構成を読み取りながら作業できる。
- **マークダウン（.md）が基本**で、AIとの相性が良い。
- **日本語化**: `Cmd + Shift + P` →「Configure Display Language」→ 日本語選択 → 再起動。
- **推奨モデル**: Claude Opus 4.5（プロプランで利用可能）。
- フォルダをワークスペースに追加すると、その中身を参照しながらAIと作業できる。

### Cursor設定の要点（#1資料より）

| 項目 | 内容 |
|------|------|
| 日本語化 | コマンドパレットで「Configure Display Language」→ 日本語 |
| セキュリティ | 設定で「Data Privacy」を有効化（学習利用オフ） |
| AIモデル | チャット下部のモデルメニューでClaude / GPT-4o / o3等を切り替え |
| @参照 | `@`でファイル・Webを指定して文脈に含める |
| Tavily Search | MCPでAPIキー登録すると高速・高精度検索が可能 |
| .cursor/rules | プロジェクト内に`.mdc`で執筆ルールを固定 |
| 自動補完 | 灰色の提案を`Tab`で確定 |
| クイックエディット | 範囲選択 → `Cmd + K`でその場でAIに書き換え |
| チャット | `Ctrl + L`で開始、`Ctrl + W`で閉じる |

## 3. Cursorの実用例：メール自動送信システム

- **空室対策の個別管理方式**でのフォローメール送信を自動化。
- Excelの管理会社リストとマークダウンのメール文面を使い、**Gmail API**でBCC一括送信を実現。
- プランモードで日本語で会話しながらプログラムを作成し、エラーも自動修正。
- API設定も**AIに聞きながら進められる**ため、詳しい知識がなくても実装可能。

---

## 4. NotionとObsidianの比較

- **当面はNotionを共有ツールとして使用**する方針で決定。
- **Notion**: 共有がしやすく、チームの情報共有に適している。CursorはMCPでNotionに接続可能。
- **Obsidian**: ローカルでAI連携しやすいが共有が難しいため、個人利用向き。外部情報の取り込み（Kindle、Webクリップ等）が得意。

---

## 5. 共有フォルダの設定

- **Googleドライブ**に共有フォルダ「DX互助会」を作成。
- フォルダ構成（議論時案）: 01_Meeting（議事録）、02_Projects、03_Resources、04_Templates、05_Knowledge。
- **共有フォルダをCursorに接続する方法**: Googleドライブでショートカットをマイドライブに作成し、そのフォルダをCursorのワークスペースに追加。

※現在の共有フォルダは 01_prompts, 03_outputs, 04_logs, 05_knowledge, 99_archive 等で運用。

## 6. その他の有用なツール紹介

### Google Apps Script（GAS）
- Gmail情報を自動取得してスプレッドシートに出力。
- トリガーで定期実行（例: 毎日特定時刻にメール取得）。
- Google Maps APIと連携して管理会社情報を自動収集。

### Genspark（ジェンスパーク）
- 定期タスクの自動実行が可能。
- 融資情報メールを週次でまとめてNotionに自動投稿する事例あり。
- AIエージェントでワークフローを組める。

### Alfred（Mac限定）
- ランチャーで作業効率を向上。`Cmd + スペース`で全ファイル検索。
- スニペットで定型文を短縮入力。クリップボード履歴（`Option + Cmd + V`で過去7日分）。
- ワークフローで住所入力→ハザードマップ・地価・人口統計を一括表示等。

### Claude Code Work
- ローカルファイルを直接参照して作業。Macアプリ版のみ（有料プラン必要）。
- ファイル名変更・フォルダ整理をリアルタイムで実行。CursorよりシンプルなUI。

### 音声入力
- **Aqua Voice**: Macでファンクションキー押下で高精度文字起こし。
- **Typeless**: スマホでも利用可能で、通勤時間などの活用に適する。

## 7. セキュリティと留意点

- 神大家クラブのコンテンツは**スクレイピング・データベース化に慎重**に。個人利用は問題ないが外部流出リスクに注意。
- **会社支給PCでの利用は避ける**（特に権限の広いツール）。
- AIツールへの過度な依存について、木村先生から懸念の声あり。
