# 東海労金 インターネットバンキングでログイン・振込を行う

このコマンドは **次の3ステップ** で動く。

---

## ステップ 1: ログインIDとパスワードの確認（コマンド実行直後）

- コマンドが実行されたら、まず認証情報を確認する。
- **「東海労金のログインIDとパスワードは .env に設定済みですか？未設定の場合は、このチャットで渡すか、browser_automation の .env に TOKAIROKIN_USER と TOKAIROKIN_PASS を追加してください。」** と案内する。
- ユーザーがチャットで ID/パスワードを渡した場合は、その場で環境変数として扱い、一時的に .env に追記するか、実行時に渡す方法をとる（セキュリティに配慮し、チャット履歴に残さない運用を推奨）。

---

## ステップ 2: 振込内容の確認

- **「振込先・振込金額・振込日を教えてください」** と聞く。
- ユーザーが回答したら、その内容を把握する。

---

## ステップ 3: ログイン → 振込の実行

ユーザーから認証情報と振込内容の回答を受けたら、次の順で実行する。

1. **スクリプトの実行**
   - スクリプトは **215_神・大家さん倶楽部** 側にある。次の絶対パスをカレントにして実行する。
   - カレント: `.../OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/browser_automation/`（ユーザーの環境では `Library/CloudStorage/OneDrive-個人用/215_神・大家さん倶楽部/C1_cursor/browser_automation` が含まれるパス）
   - 仮想環境を有効化してから `python fetch_after_login.py tokairokin` を実行する（または東海労金専用スクリプトがあればそれを実行）。
   - **未セットアップの場合**は、215 の browser_automation README を参照し、東海労金用の config_tokairokin.yaml と .env の設定を案内する。

2. **ログインURL**
   - 東海労金インターネットバンキング: https://ib.rokin.jp/nprotect/?bid=22

3. **振込の実行**
   - スクリプトがログイン後、振込画面へ遷移し、ユーザーが指定した振込先・金額・日付を入力して振込を実行する。
   - 初回は `headless: false` のまま実行し、ブラウザでログイン・振込が成功するか確認するよう促す。

---

## 注意

- ログインに失敗している場合は、その旨を伝え、headless: false で画面確認するよう促す。
- 認証情報（.env）は 215 の browser_automation にあり、リポジトリに含めずユーザー自身で設定する。
- **振込は金銭を移動する操作のため、実行前に振込内容をユーザーに確認させる。**
