## 2-1-5 契約書の4視点チェック

```
あなたは契約書分析の専門アシスタントです。

添付した契約書を、私（乙（受託者））の視点から４つの観点で整理してください。

① 自分から見たメリット
② 自分から見たデメリット
③ 相手から見たメリット
④ 相手から見たデメリット

最後に注意ポイント（あいまいな表現や確認すべき項目）があれば記載してください。
```
#### 絶対に守るべき注意点（AIに聞いていいこと・悪いこと）

![画像](https://ptx-media-storage.imgix.net/raimo/2-1-5_03.jpeg)

ここは非常に重要なので、しっかり理解してください。

  

AIに聞いてよいのは、「この契約書のメリットとデメリットを整理して」という内容の整理・分析です。これは契約書に書かれている内容を読み取って分類する作業なので、AIが事実を読み間違えるリスクが比較的低い領域です。

  

一方、AIに聞いてはいけないのは、「この契約書は法的に問題ないか？」「この条項は有効か？」といった法的判断です。なぜなら、法律の解釈にはAIのハルシネーション（事実でないことをもっともらしく回答してしまう現象）のリスクが高いからです。AIが存在しない法律を「ある」と言ったり、最新の法改正を反映していなかったりする可能性があります。

  

|AIに聞いてOK|AIに聞いてはNG|
|---|---|
|この契約のメリット・デメリットは？|この契約は法的に有効か？|
|自分に不利な条項はどれか？|この条項は違法ではないか？|
|一般的な契約と比べて厳しい点は？|この契約で訴えられる可能性は？|
|見落としている論点はないか？|法的なリスクはどの程度か？|

法的な判断が必要な重要案件は、必ず弁護士に相談してください。AIの4視点チェックは「見落とし防止ツール」であり、法律の専門家の代わりにはなりません。

#### 温度感（トーン）の指定方法

メールの悩みの多くは「何を書くか」ではなく「どう書くか」——つまり温度感（トーン）の問題です。AIには、この温度感を言葉で指定できます。

  

|温度感の指定例|AIが出力する文体のイメージ|
|---|---|
|「丁寧かつ誠実な感じで」|フォーマルだが温かみのある文体|
|「共感を込めた柔らかいトーンで」|相手の気持ちに寄り添う表現|
|「ビジネスライクに簡潔に」|要件のみの短い文体|
|「角が立たないように」|婉曲的で配慮のある表現|
|「少しカジュアルだけど失礼のないように」|親しみやすいが礼儀正しい文体|
|「強めの催促だが関係を壊さない程度に」|毅然としつつも相手を尊重する表現|

ポイントは、正確な用語を使う必要がないことです。「丁寧だけど堅すぎない感じ」「怒ってはいないけど困っている感じ」のように、自分の感覚をそのまま伝えればAIは汲み取ってくれます。音声入力を使えば、頭に浮かんだニュアンスをそのまま伝えられるので特に相性がいいです。

```
以下は{相手の立場}からの{メッセージの種類}です。
この内容を理解してください。相手の意図やトーンも含めて把握してください。
理解が完了したら「理解しました」とだけ回答してください。
この後、返信案の作成を依頼します。

{相手のメッセージ}
```

```
先ほどの{メッセージの種類}に対する返信を{パターン数}パターン作成してください。

【返信の方向性】
{伝えたい内容}

【温度感の指定】
{温度感}

【条件】
・それぞれ異なるアプローチ（切り口や表現）で作成してください
・各パターンに「このパターンの特徴」を1行で添えてください
・長さは{長さの目安}程度にしてください
```

#### 変数の説明

|変数名|入力する内容|入力例|
|---|---|---|
|{メッセージの種類}|何に対する返信か|メール、LINEメッセージ|
|{パターン数}|出力する案の数|3|
|{伝えたい内容}|返信で伝えたいことの要点|お礼を伝えつつ、納期を1週間延ばしてもらえないか相談したい|
|{温度感}|文章のトーン|丁寧かつ誠実な感じで、無理なお願いをしている申し訳なさも出す|
|{長さの目安}|希望する文章の長さ|200文字|

## 7-2-4 編集
### 1. 生成AIによる構成
### 2. 論理構成のチェック
### 3. トーンの統一
#### 編集フェーズの5つの作業と担当

![画像](https://ptx-media-storage.imgix.net/raimo/7-2-4-02.jpeg)

|ステップ|作業|担当|理由|
|---|---|---|---|
|①校正|誤字脱字・文法ミスのチェック|AIに任せてOK|パターン認識が得意|
|②論理構成チェック|流れに矛盾・飛躍がないか|AIに任せてOK|論理的整合性の確認が得意|
|③トーン・表現の統一|全体の一貫性を確認|AIに任せてOK|統一性のチェックが得意|
|④事実確認|固有名詞・数字・日付の正確性|人間が確認|AIは事実の正誤を判断できない|
|⑤最終判断|これで公開・提出していいか|人間が判断|責任を持つのは人間|

出版社に例えると、AIは「優秀な校閲者」。文法のミスや論理の飛躍は見つけてくれるが、「この人名で合っていますか？」「この数字の出典は？」という確認は著者本人（人間）にしかできない。

  

#### なぜこの順序で進めるのか

①校正→②論理→③トーンの順番には理由がある。まず表面的なミス（誤字脱字）を潰してから、構造的な問題（論理の飛躍）を確認し、最後に全体の印象（トーン統一）を仕上げる。逆にすると、トーンを整えた後に論理の修正で文章が変わり、再度トーンチェックが必要になる。

プロンプト1：ステップ①：校正チェック

#### プロンプトの目的

文章の誤字脱字・文法ミス・不自然な言い回しを一括でチェックし、修正候補を一覧で出させる。

  

#### プロンプト本文

コード

```
以下の文章を校正してください。

チェック項目：
- 誤字脱字
- 文法の誤り
- 不自然な言い回し
- 同じ語尾の連続（3回以上）

修正が必要な箇所は「原文→修正案」の形式で一覧にしてください。
修正不要の場合は「問題なし」と記載してください。

---
{校正対象の文章}
```

プロンプト2：ステップ②：論理構成チェック

#### プロンプトの目的

文章の論理の流れに飛躍や矛盾がないかをチェックし、問題箇所と改善案を出させる。

  

#### プロンプト本文

コード

```
以下の文章の論理構成をチェックしてください。

確認項目：
- 話の流れに飛躍がないか
- 主張と根拠が対応しているか
- 結論が唐突になっていないか
- 読者が「なぜ？」と疑問に思う箇所がないか

問題がある箇所は、該当部分・問題の内容・改善案をセットで教えてください。
問題がない場合は「論理構成に問題なし」と記載してください。

---
{チェック対象の文章}
```

プロンプト3：ステップ③：トーン・表現の統一チェック

#### プロンプトの目的

文章全体を通してトーンや表現に不統一がないかを確認し、修正が必要な箇所を指摘させる。

  

#### プロンプト本文

コード

```
以下の文章全体を通して、トーンや表現に
統一されていない箇所がないか確認してください。

チェック項目：
- 敬体（です・ます）と常体（だ・である）の混在
- 硬い表現と柔らかい表現の混在
- 同じものを違う言葉で呼んでいる箇所
- セクション間でのトーンの変化

不統一な箇所があれば、該当部分と統一案を教えてください。
問題がない場合は「トーン統一に問題なし」と記載してください。

---
{チェック対象の文章}
```
#### 人間が必ず確認すべき5つの項目

|項目|確認すること|なぜAIに任せられないか|
|---|---|---|
|固有名詞の正確性|社名・人名・商品名が正しいか|AIは「もっともらしい間違い」をする|
|数字・日付の正確性|金額・件数・期日が正しいか|AIは数字を平気で間違える|
|社内ルールとの整合性|用語・表記ルールに合っているか|AIは社内ルールを知らない|
|機密情報の有無|うっかり含まれていないか|AIは機密の判断ができない|
|最終的な公開判断|これで出していいか|責任を持つのは人間|

特に固有名詞と数字は要注意。AIは「それらしい名前」「それらしい数字」を自信を持って出力するため、間違いに気づきにくい。必ず原本（元データ、公式サイトなど）と照合する。

  

#### 音読チェックの方法

ステップ④の事実確認が終わったら、最後に文章を声に出して読む。黙読では気づかない不自然さが、音読すると浮き上がる。

  

チェックのポイント：つっかえる箇所がないか（読みにくい＝分かりにくい）、同じ語尾が続いて単調になっていないか、前置きが長くて本題がなかなか始まらない箇所がないか。
