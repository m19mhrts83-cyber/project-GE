## Shinさんのフォルダ構成（ジェネトピ動画より）

出典: ジェネトピ「Obsidian × Cursorで思考を拡張するAIを構築してみた。」
https://note.com/genai_topic/n/n5a663fd8016d

```
Vault/
├── Journaling/         … 日々の気づきや走り書きのメモ
├── Clippings/          … Web記事など外部の情報を保存（Web Clipper活用）
├── Literature Note/    … Kindleのハイライトなど、読書から得た知識を保存
└── Permanent Note/     … 自分のオリジナルの考えやアイデアをまとめる場所
```

### 各フォルダの役割

| フォルダ | 目的 | 具体的な内容 |
|----------|------|--------------|
| Journaling | 日々のインプット | 日記、気づき、走り書きメモ |
| Clippings | 外部情報の収集 | Web Clipperで保存した記事、SNS投稿など |
| Literature Note | 読書の知識化 | Kindleハイライトの自動同期、読書メモ |
| Permanent Note | 自分の思考の結晶 | 情報を咀嚼して自分の言葉でまとめたノート |

### Shinさんのメモ構造化フレームワーク

Excalidrawプラグインで図解を作成し、以下のフレームワークでメモを構造化している：

1. **Fact（事実）** … 読んだ・聞いた・体験した事実をそのまま記録
2. **Abstraction（抽象化）** … 事実から汎用的な原則や法則を抽出
3. **Application（転用）** … 抽象化した知見を自分の状況に当てはめる

### 情報の流れ

```
情報収集 → Clippings / Literature Note に保存
    ↓
日々の気づき → Journaling に記録
    ↓
咀嚼・再構成 → Permanent Note にまとめる
    ↓
CursorにVaultごと読み込ませ、AIの「コンテキスト」として活用
```

## 他によく紹介されるフォルダ構成例

### パターン1: Zettelkasten（ツェッテルカステン）方式

Odysseas氏が紹介する「思考のライフサイクル」に沿った構成：

```
Vault/
├── 00 Inbox/            … 思いついたことを即座にキャプチャする仮置き場
├── 01 Rough Notes/      … ラフに書き出したメモやアイデア（未整理OK）
├── 02 Source Material/  … 書籍・Web記事・ポッドキャストからの引用や読書メモ
├── 03 Permanent Notes/  … 自分の言葉で再構築した完成ノート
├── 04 Projects/         … 実務・学習・執筆など目的を持った活動単位
└── Templates/           … テンプレートを保存してノート作成を高速化
```

### パターン2: シンプル方式（がんばらないObsidian）

必要最低限のディレクトリ構成で始める方法：

```
Vault/
├── Config/        … 設定関連
│   ├── Extra/     … 画像などの添付ファイル
│   └── Templates/ … ノートの雛形
├── Daily/         … デイリーノート
├── Notes/         … デイリーノートから切り出したノート
└── Outputs/       … 用途別ノート（Qiita、仕事など）
```

### 共通するポイント

- **フォルダは深くしすぎない**（Obsidianはリンクで整理するのが基本思想）
- **Inboxを必ず用意する**（「まず書く」ハードルを下げるため）
- **最終的にPermanent Noteへ昇華させる**のがゴール
- **テンプレートフォルダ**を用意して記録の標準化・高速化を図る
