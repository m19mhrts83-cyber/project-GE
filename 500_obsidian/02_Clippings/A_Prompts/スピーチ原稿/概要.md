LESSON 1

聴き手視点で逆算する ― 聴き手分析とゴール設定

このレッスンの目的

スピーチの出発点を「何を話すか」から「聴き手にどう届くか」に転換し、聴き手の分析とゴール設定ができるようになる。

このレッスンで学ぶこと

- 9-2-1のインタビュー型との違い（AIの使い方が変わる理由）
- 聴き手を徹底的に理解する方法
- ゴールを先に決める「3つのゴール設定」

解説

### 9-2-1のインタビュー型との違い



前のレッスンでは、AIを「インタビュアー」として使い、自分の中にある素材を引き出しました。スピーチ原稿でもこの手法は使えますが、スピーチにはもう1つ重要な要素があります。「聴き手の反応」です。

企業理念は書いたら完成ですが、スピーチは聴き手の頭の中で完成します。同じ原稿でも、聴き手によって届き方が違う。だからこの講座では、AIの役割が「質問する人」から「聴き手を演じる人」に変わります。


| 比較項目  | インタビュー型（9-2-1） | 聴き手視点逆算型（この講座）   |
| ----- | -------------- | ---------------- |
| AIの役割 | 質問して素材を引き出す    | 聴き手を演じてフィードバックする |
| 出発点   | 自分の想いやエピソード    | 聴き手にどう届くか        |
| 品質の判断 | 読み手視点チェック      | 聴き手役によるシミュレーション  |


### 聴き手を徹底的に理解する

スピーチ作成の第一歩は、聴き手を深く理解することです。


| 項目           | 問いかけ            |
| ------------ | --------------- |
| 誰が聴くのか       | 年齢層、立場、人数は？     |
| どんな状態で聴くのか   | 期待している？義務で来ている？ |
| 何を知っているのか    | 前提知識はどの程度？      |
| 何を期待しているのか   | 聴き手が聞きたいことは？    |
| 何に不安・疑問があるのか | 解消してほしい懸念は？     |


### ゴールを先に決める「3つのゴール設定」

構成を考える前に、スピーチが終わった時に聴き手がどうなっていてほしいかを先に決めます。


| ゴールの種類 | 問いかけ           | 例                  |
| ------ | -------------- | ------------------ |
| 感情ゴール  | どんな気持ちになってほしいか | 温かい気持ち、やる気が出る、安心する |
| 理解ゴール  | 何を理解してほしいか     | 新プロジェクトの意義、方針転換の理由 |
| 行動ゴール  | 何をしてほしいか       | 明日から〇〇を始める、協力を申し出る |


すべてのゴールを設定する必要はありません。スピーチの種類によって、感情ゴールが中心（結婚式）、理解ゴールが中心（プレゼン）など重点が変わります。

### 比喩・例え話

スピーチ原稿を一人で推敲するのは、自分の料理を自分で味見するようなもの。味覚が慣れてしまって塩加減が分からなくなります。AIに聴き手役を演じさせるのは「初めて食べる人」に試食してもらうのと同じ。新鮮な感想がもらえます。

LESSON 2

構成設計と下書き生成

このレッスンの目的

聴き手の心の動きに沿ったスピーチ構成を設計し、下書きを生成できるようになる。つかみと締めの磨き方も学ぶ。

このレッスンで学ぶこと

- 聴き手の心の動きに沿った5パート構成
- 下書きの生成方法
- つかみと締めの磨き方
- 時間と文字数の目安

解説

### 聴き手の心の動きに沿った5パート構成



スピーチの構成は、聴き手の心が動く順番に沿って設計します。


| パート       | 聴き手の心理      | 役割          |
| --------- | ----------- | ----------- |
| ① つかみ     | 「お、面白そう」    | 関心を引く       |
| ② 共感      | 「分かる分かる」    | 聴き手との接点を作る  |
| ③ 本題      | 「なるほど」      | 伝えたいことを届ける  |
| ④ クライマックス | 「じーん…」「よし！」 | 感情を動かす      |
| ⑤ 締め      | 「いいスピーチだった」 | 印象を残す・行動を促す |


### 場面別の型一覧

スピーチの型は「聴衆に何を持ち帰ってほしいか」で選びます。場面に合った型を選ぶことが大切です。


| 場面       | おすすめの型 | 型の特徴                | 構成イメージ              |
| -------- | ------ | ------------------- | ------------------- |
| 朝礼・会議の一言 | PREP型  | 結論ファーストで情報を端的に伝える   | 結論→理由→具体例→結論        |
| プレゼン・提案  | PREP型  | 論理的に理解・納得させる        | 結論→理由→事例→結論＋行動喚起    |
| 結婚式・送別会  | エピソード型 | 具体的な思い出で感情を動かす      | エピソード→気づき→メッセージ     |
| 式典・就任挨拶  | ストーリー型 | 過去→現在→未来の流れでビジョンを語る | 背景→転機→現在→未来への決意     |
| セミナー・講演  | 課題解決型  | 聴衆の課題を提示し解決策を示す     | 課題提起→原因→解決策→行動喚起    |
| 表彰・お祝いの場 | エピソード型 | 相手のエピソードで称賛・祝福する    | エピソード→その人の価値→お祝いの言葉 |


### 時間と文字数の目安

スピーチの文字数は「1分≒300文字」が目安です。ゆっくり話す場合は250文字程度。


| 持ち時間  | 文字数目安          | よくある場面         | パートの配分目安                            |
| ----- | -------------- | -------------- | ----------------------------------- |
| 1分    | 約250〜300文字     | 朝礼の一言、乾杯の挨拶    | つかみ20%・本題60%・締め20%                  |
| 3分    | 約750〜900文字     | 祝辞、短い挨拶、表彰コメント | つかみ15%・共感15%・本題40%・クライマックス20%・締め10% |
| 5分    | 約1,250〜1,500文字 | 式典スピーチ、来賓挨拶    | つかみ10%・共感15%・本題40%・クライマックス25%・締め10% |
| 10分   | 約2,500〜3,000文字 | セミナー冒頭、講演      | つかみ10%・共感10%・本題45%・クライマックス25%・締め10% |
| 15分以上 | 約3,750文字〜      | 基調講演、研修        | つかみ5%・共感10%・本題50%・クライマックス25%・締め10%  |


最も多い失敗は時間オーバーです。早口で詰め込むのは厳禁。間を取りながらゆっくり話す前提で文字数を設計してください。

### 書き言葉→話し言葉の変換ルール一覧

スピーチ原稿は「聴いて分かる」話し言葉で書く必要があります。以下のルールで変換してください。


| ルール            | 書き言葉（変換前）        | 話し言葉（変換後）        | ポイント              |
| -------------- | ---------------- | ---------------- | ----------------- |
| 一文を短くする        | 一文60文字以上の長い文     | 一文40文字以内に区切る     | 長い文は耳で追えない        |
| 接続詞を口語にする      | しかしながら、したがって、および | でも、だから、そして       | 硬い接続詞は耳に残らない      |
| 数字を聴いて分かる表現にする | 127.3%増加、36,542名 | 約3割増し、約3万6千人     | 細かい数字は聞き取れない      |
| 漢語を和語にする       | 遂行する、懸念される、促進する  | やり遂げる、心配される、進める  | 漢語は同音異義語と混同しやすい   |
| 「間」を入れる        | 文がずっと続く          | 大事な一言の前後に（間）を入れる | 間があると聴衆が考える時間ができる |
| 同音異義語を避ける      | 「以降」の取り組み        | 「これから先」の取り組み     | 「以降」と「移行」が区別できない  |
| 主語を明確にする       | （主語省略）実施しました     | 私たちが実施しました       | 聴いているだけでは主語を補えない  |


### つかみと締めの磨き方

スピーチで最も印象に残るのは最初と最後です。つかみにはいくつかのパターンがあります。


| パターン  | 効果       | 例                  |
| ----- | -------- | ------------------ |
| 質問    | 参加意識を高める | 「皆さん、〇〇したことありますか？」 |
| 意外な事実 | 好奇心を刺激する | 「実は今日、私は〇〇なんです」    |
| エピソード | 情景が浮かぶ   | 「先日、こんなことがありました」   |
| 共感    | 一体感を作る   | 「〇〇って、大変ですよね」      |


締めはゴールに直結する部分です。感情ゴールなら心に残るメッセージ、行動ゴールなら具体的な呼びかけで締めます。

LESSON 3

AIに聴き手役を演じさせてフィードバックをもらう

このレッスンの目的

AIに聴き手役を演じさせてスピーチ原稿のフィードバックをもらい、複数の視点で多角的にチェック・修正する方法を習得する。これがこの講座の核心。

このレッスンで学ぶこと

- AIに聴き手役を演じさせるフィードバックの方法
- 複数の聴き手視点で多角チェックする意味と方法
- フィードバックを整理して原稿を修正する流れ

解説

### AIに聴き手役を演じさせるフィードバック

ここが本講座の核心です。下書きができたら、AIに「あなたはこのスピーチの聴き手の一人です」と役割を与え、聴いた立場での正直な感想をもらいます。

ポイントは、聴き手役の「設定」を具体的にすることです。「聴き手として評価して」だけでなく、年齢、立場、このスピーチへの期待度、気になっていることまで設定すると、よりリアルなフィードバックが返ってきます。

### 複数の聴き手視点で多角チェック



スピーチの聴き手は一人ではありません。同じスピーチでも、立場によって受け取り方が異なります。

例：結婚式スピーチの場合

- 1回目：新婦の親友として → 「二人の思い出が伝わって嬉しい」
- 2回目：新郎のご両親として → 「内輪ネタが多くて少し置いてけぼり感」
- 3回目：あまり親しくない会社関係者として → 「長い…いつ終わるのか」

このように、ある聴き手には好評でも別の聴き手には不評な部分が見つかります。これが多角チェックの価値です。

### フィードバックを整理して修正する

複数のフィードバックをもらったら、「共通して良い点」「共通して改善が必要な点」「意見が分かれた点」を整理します。共通の改善点は必ず修正し、意見が分かれた点はメインの聴き手に合わせて判断します。

LESSON 4

スピーチ種類別テンプレートと最終チェック

このレッスンの目的

よくあるスピーチの種類ごとにテンプレートを理解し、最終チェックの方法を習得する。

このレッスンで学ぶこと

- スピーチ種類別テンプレートの使い分け
- 本番前の最終シミュレーション

解説

| 朝礼スピーチ  | 社員         | 感情：やる気、気づき    | 1〜3分   |

種類によって重点が変わります。結婚式や送別会は感情ゴールが中心、プレゼンは理解＋行動ゴールが中心です。

### 本番前の最終シミュレーション

原稿が完成したら、本番想定で最終チェックを行います。

### 声に出して確認する重要性

スピーチ原稿は「読む文章」ではなく「話す文章」です。AIのチェックに加えて、必ず声に出して読んでください。確認のポイントは以下の通りです。

- 制限時間内に収まるか（1分≒300文字で計算）
- 息継ぎがしにくい長い文はないか
- 「聴いて」分かりにくい同音異義語はないか（例：「以降」と「移行」）
- 口に出しにくい言い回しはないか

講座のまとめ

よくある疑問と回答

スピーチが苦手でも使えますか？

はい。むしろスピーチが苦手な人ほど効果的です。「何を話せばいいか分からない」という悩みは、聴き手分析とゴール設定で解消されます。原稿自体はAIが生成してくれるので、あなたがやるのは「誰に、どう届けたいか」を考えることだけです。

AIに聴き手役をさせたフィードバックは信頼できますか？

100%正確ではありませんが、一人で推敲するよりも確実に多くの気づきが得られます。特に「自分が気づかない視点」を提供してくれる点に価値があります。最終判断は自分でしてください。

即興スピーチ（準備なし）にも使えますか？

完全な即興には間に合いませんが、朝礼の一言スピーチのように「テーマだけ決まっていて数分の準備時間がある」場合は、ゴール設定と構成設計のプロンプトだけでも十分活用できます。

上司のスピーチ原稿を代筆する場合はどうすればいいですか？

2つのステップで対応できます。まずプロンプト3（AIインタビュアー）の質問リストを使って上司にヒアリングし、回答をもとに原稿を生成します。次にプロンプト5（話し言葉変換）で上司の話し方の特徴（口癖、話すスピードなど）を指定して口調を調整します。最後は必ず本人に確認してもらってください。

この講座で理解したこと

1.スピーチは「聴き手にどう届くか」で成否が決まり、ゴールから逆算して構成を設計する

2.AIに聴き手役を演じさせることで、本番前にスピーチの反応をシミュレーションできる

3.複数の聴き手視点でチェックすることで、一つの視点では気づけない改善点が見つかる

次のレッスンへの橋渡し

スピーチ原稿で「聴き手視点逆算型」を学びました。次のレッスン「求人票の作成」では、9-2-1のインタビュー型と今回の読み手チェックを組み合わせた「参考求人＋AIインタビュー型」を学びます。参考求人から項目を自動抽出し、AIの質問に答えるだけで求人票が完成。さらに求職者視点でチェックするという、これまで学んだ手法の応用編です。
