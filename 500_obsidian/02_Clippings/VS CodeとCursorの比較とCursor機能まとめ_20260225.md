# VS Code と Cursor の比較・Cursor 機能まとめ

確認日: 2026-02-25

---

## 1. 先ほど調べた内容

### 1.1 キーバインド（KeyBind）

- **キーバインド** = 「このキー（の組み合わせ）を押したら、この操作を実行する」という割り当て。キーボードショートカットと同じ意味。
- **メリット**: マウス不要で素早く操作できる、手をキーボードから離さない、自分好みに変更できる。
- **表記**: Cmd（Mac）≒ Ctrl（Win）、Shift / Alt / Enter は共通。
- **VS Code・Cursor での開き方**: `Cmd + K` → `Cmd + S`（Mac）、または 歯車 → Keyboard Shortcuts。ここで一覧・検索・変更が可能。

### 1.2 VS Code でドキュメント・文章作成（AI なし／ChatGPT 併用）

- **AI なし**: Markdown で見出しを付けて構成メモ、アウトラインビューで整理、スニペットで定型文（メール頭など）を挿入、複数ファイルを並べて編集。プレビューは `Cmd + Shift + V`。
- **ChatGPT と併用**: (A) VS Code で構成・要点を書き → コピーして ChatGPT で文案生成 → 返答を VS Code に貼る。(B) 下書きを VS Code で書き → 該当部分を ChatGPT で推敲・要約 → 貼り戻す。(C) 画面を半分に分けて VS Code とブラウザの ChatGPT を並行利用。
- **役割分担**: 「書く・編集・構成 = VS Code」「生成・推敲 = ChatGPT」とすると、Cursor ほど統合されていなくても同様の作業が可能。

## 2. VS Code の概要と Cursor の概要

### 2.1 VS Code（Visual Studio Code）

- **開発元**: Microsoft
- **位置づけ**: 汎用のコードエディタ。拡張機能（Extension）で言語サポート・AI（例: GitHub Copilot）などを追加する前提。
- **強み**:
  - 拡張機能の数・情報量・企業での利用実績が豊富
  - 無料で個人・企業利用可能。ライセンス・テレメトリの説明がしやすい
  - Live Share（共同編集）、リモート開発（SSH / WSL / Dev Container）などが充実
- **AI**: 標準では「エージェントがファイルを編集する」機能はなし。Copilot 等の拡張で補完・チャット程度は可能。

### 2.2 Cursor

- **開発元**: Cursor Inc.
- **位置づけ**: **VS Code をベースにした「AI ファースト」のエディタ**。Composer（エージェント）、Chat、複数 AI モデル選択、コードベース索引などが最初から組み込まれている。
- **強み**:
  - チャットや Composer で「この条件でメール文案を」「発表の構成を」と依頼して生成・編集できる
  - 複数モデル（Claude / GPT 等）を選べる
  - 見た目・キーバインドは VS Code に近く、移行がしやすい
- **注意**: コード・文章は利用する AI プロバイダの API に送信される。仕事で使う場合は社内ポリシー確認が必要。

## 3. Cursor の機能についてのまとめ

※ C1_cursor・C2_ルーティン作業フォルダ内の運用を基準に整理しています。

### 3.1 今まで使った機能

- **Rules（.cursor/rules）**: 215 などに `git-commit-habit.mdc`（コミットは促すが代行しない）、`partner-email-check.mdc`（「パートナーからのメールを確認して」で gmail_to_yoritoori / imessage_to_yoritoori 実行）、`gmail-token-troubleshooting.mdc`（token 更新時の連携案内）を配置済みで運用している。
- **チャット（AI との会話）**: 質問・相談・文章の推敲・要約・構成案の依頼を日本語で行う。
- **チャット + @ファイル**: `Cursorチャット文例_メール送信.md` のように、`@MDファイル` と `@送信先一覧.xlsx` を指定してメール送信まで依頼。空室対策メールのテスト送信・本番送信で利用。
- **Composer（エージェント）**: メール送信スクリプトの実行、パートナー確認（「パートナーからのメールを確認して」）、「やり取りのMDファイルへ履歴として追加して。そして Notion の所有物件タスク管理へ新規プロジェクト登録して」などのトリガーで複数ステップを依頼。
- **MCP の利用**: Notion（タスク登録。create-task は parent バグで不安定なため `create_notion_task.py` を併用）、Gmail（mcp.json 設定済み。送信は主に Python スクリプト）。MCP 設定・エラー解消は「AI に依頼しながら進める」運用（`260211_MCP設定手順.md`）。
- **ターミナル実行の依頼**: `gmail_to_yoritoori.py` / `imessage_to_yoritoori.py`、空室対策メール送信スクリプトを Cursor に実行させる。
- **ドキュメント・文章作成**: メール文案、送信下書き、やり取り.md への追記、空室対策メール本文の作成を Cursor 内で依頼。
- **インライン補完（Tab 補完）**: 入力の続きを AI が提案し、Tab で受け入れる。

### 3.2 まだ使っていないが、便利そうで役立ちそうな機能（C1_cursor・C2 内で言及されているもの中心）

- **Gmail MCP の本格活用**: 現在は `gmail_to_yoritoori.py` でやり取り.md 追記が中心。MCP の Gmail は「チャットから Gmail を検索・操作する」用途で設定はあるが、まだ使いこなしていない。`260211_MCP設定手順.md`・`Gmail token 更新時の連携（いけとも・パートナー）.md` 参照。
- **Notion MCP の create-task / create-page の安定利用**: `Notion_MCP_parent文字列化バグ_調査報告.md` のとおり、モデルを Sonnet/Opus に固定すると MCP が正常に動く。スクリプト代替に頼らず、チャットから「Notion にタスクを1件追加して」と依頼する運用を試すとよい。
- **Notion の他の MCP スキル**: タスク登録以外（データベース検索・ページ作成・会議準備・リサーチドキュメント作成など）は C1/C2 内では未活用。Notion に溜まった情報の検索・要約に使える。
- **Tavily MCP**: Web 検索。`260211_MCP設定手順.md` に設定あり。管理会社候補の調査や問い合わせ文の下調べなどに使えそう。
- **入居者募集案内の半自動生成**: `プロンプト_入居者募集案内自動化.md` は実装経緯の記録。物件ごとに「この物件で募集案内の文案を出して」と Cursor に依頼する運用を習慣化すると、`日々のルーチンワークと面倒な作業一覧.md` の「今すぐ着手」に合う。
- **物件募集資料（チラシ）の半自動生成**: 同様に「30分→5分」を目標に、テンプレと Cursor で毎回のチラシ文面を生成する流れがまだ。
- **相談文・依頼文の自動生成**: 運営・税理士・管理会社への相談文を「依頼内容の要点」だけ書いて Cursor に文案生成させる（文章作成時間削減）。
- **Cursor 標準フォルダ構成の導入**: 「あの資料どこだっけ？」解消のため、`日々のルーチンワーク…` の優先リストにある。フォルダ構成のルールを決めて Rules や @フォルダ で参照しやすくする。
- **Agent の複数ステップ一括実行の習慣化**: 「やり取り追記＋Notion 登録」はルールでトリガー済み。プロジェクト完了時の「送信添付を過去フォルダへ移動」など、一連作業を「〇〇して、そのあと〇〇して」とまとめて依頼する使い方はまだ増やせそう。
- **複数モデルの使い分け**: Notion MCP は Auto だとバグ、Sonnet/Opus で正常。軽い編集は速いモデル、Notion 操作や難しい構成は強いモデルと意図的に切り替えると効く。
- **キーバインドのカスタム**: よく使う操作（例: メール送信文例を開く、やり取り.md を開く）を自分用のショートカットに割り当てると、ルーティンが速くなる。
